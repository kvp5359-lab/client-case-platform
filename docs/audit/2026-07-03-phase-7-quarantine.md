## Резюме

Карантинный слой функционально здоров и активно используется всеми 5 каналами в проде (telegram 5066 сообщений, mtproto 3876, email 935, business 290, wazzup 132). Прежние IDOR (`telegram-edit-message`, `telegram-send-message`) исправлены. mtproto-контейнер стабилен (0 рестартов, uptime 7 дней, только безобидные ошибки аватарок). Но главный риск карантина — **не баги, а огромный накопленный хвост незадеплоенных и несмоканных изменений**: журнал расследований полон записей «⏳ ЖДЁТ ДЕПЛОЯ / ЖДЁТ СМОК», причём часть правок уже в проде на стороне БД (через MCP), а фронт/edge к ним — нет. Это «полу-выкаченное» состояние хрупко для продаваемого сервиса. Плюс три известных незакрытых остатка (гонка глобального токена на отправке, anon-гранты, отсутствие soft-delete в части каналов). Для нового воркспейса-покупателя подключение каналов — во многом ручное.

## Часть В — Матрица каналов и обработка ошибок

- **Матрица возможностей** (`channels.md`) — по коду правдива (проверено в аудите edge-функций батч 3): удаление по каналам разными методами (`deleteBusinessMessages` ≠ `deleteMessage`, MTProto `revoke`, Wazzup `DELETE /v3`), реакции/edit ограничения соответствуют.
- **Rate-limit (429/FLOOD_WAIT)**: после гранулярного фолбэка 06-26 основные пути отправки обрабатывают; `wazzup-delete` обёрнут в try/catch (06-01). Полной гарантии по всем путям нет — отдельные вспомогательные вызовы не проверены поштучно.
- **Email — хрупкость «микса транспортов»** (Gmail API + Resend): задокументирована (channels.md, ledger 06-29). Для ВТОРОГО покупателя (не `kvp5359@gmail.com`) онбординг email-аккаунта с нуля **не проверялся** — воспроизводимость под вопросом (см. Часть Г).

## Часть Г — Онбординг каналов для нового воркспейса (влияет на продаваемость)

Что автоматизировано / что руками при подключении каналов новому клиенту:

| Канал | Автоматизация | Ручные шаги |
|---|---|---|
| Email-домен | `provision-domain`/`provision-email-domain` (Resend + DNS) | Настройка DNS-записей на стороне клиента |

| Gmail | OAuth-флоу | Клиент проходит Google OAuth |

**Вывод Г:** значимая часть подключения каналов — ручная (боты, ключи, Premium, DNS). Для масштабной продажи это узкое место онбординга (каждый новый клиент = ручная возня владельца). Не блокер первой продажи, но ограничитель роста. Смыкается с Фазой 8.
