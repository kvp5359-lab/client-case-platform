## Находки (по убыванию эффекта)

1. **Инлайновый `onStatusChange` рушит memo всех строк** — `BoardTabContent.tsx:165` (стрелка
   пересоздаётся каждый рендер → через BoardView→BoardColumn→BoardListCard ломает
   `memo(DraggableBoardTaskRow)`). Фикс: `useCallback`.
2. **`BoardColumn`/`BoardListCard` не мемоизированы** (`BoardColumn.tsx:42`, `BoardListCard.tsx:60`)
   — каждый inbox-тик (~1.5 с) прогоняет `useBoardListCardSetup` (фильтр/группировка/сортировка)
   для всех списков. Фикс: `React.memo` после стабилизации пропов (п.1, 7).
3. **`inboxThreads` передаётся во ВСЕ списки** (`BoardView.tsx:378` → `BoardColumn.tsx:80`), хотя
   нужен только inbox-спискам → смена ссылки инвалидирует пропы всех. Передавать только inbox-спискам.
4. **Broad-инвалидация = повторный шторм**: `useCalendarThreads.ts:159,173` (drag в календаре)
   инвалидирует `calendarKeys.all` (→ рефетч 50 чанков times) и `workspaceThreadKeys.workspace`
   (→ полный рефетч ~2000 тредов → снова 50 чанков assignees). `boardFilteredKeys.threadsAll`
   инвалидируют смена статуса/дедлайна (`BoardTabContent.tsx:79`) и `useProjectThreads.mutations.ts:274`
   (карантин, факт). Одно действие с задачей = рефетч всего набора + ~100 запросов. Фикс:
   оптимистичный патч кэша вместо широкой инвалидации.
5. **Нет виртуализации** в boards/inbox-списках (grep react-window/virtuoso — пусто): 50
   `InboxChatItem` + все task-строки в DOM. Календарь DOM ограничивает сам (видимая неделя).
6. **Инлайновые per-row хендлеры `BoardInboxList.tsx:296-304`** (`onClick`/`onMarkAsRead`/…)
   ломают `memo(InboxChatItem)` → 50 строк ре-рендерятся каждый тик. Стабилизировать `useCallback`.
7. **`assignees={assigneesMap[task.id] ?? []}`** (`BoardListCard.tsx:282,314`) — новый `[]` каждый
   рендер, defeat memo. Общий `EMPTY`-литерал; то же `visibleFields ?? [...]` (:284,316).
8. **Голая подписка `useInboxThreadsV2` в `BoardTabContent:37`** (прогрев UnreadBadge): select
   флэтит страницы → новая ссылка каждые ~1.5 с → ре-рендер всей доски даже без inbox-списков.
   Гейтить/заменить на prefetch (правка в BoardTabContent, не в карантинном useInbox).
9. **Календарный список без фильтра вырождает union** (`boardFilterService.ts:101-121`):
   `get_board_filtered_threads` отдаёт ВЕСЬ воркспейс (2-3 страницы по 1000), не 1975. Фикс —
   серверный cap/не включать бесфильтровый список как «пустой» union.
10. **`taskEvents` строит ~1975 Date-пар на каждый пересчёт** (`BoardListCalendarView.tsx:106-123`),
    big-calendar сканирует все на рендер. Префильтровать events по видимому диапазону.
11. **Нет капа календарного списка** — к 5000 тредов всё линейно хуже (125 чанков ×2, 5+ страниц RPC).
    Серверно резать по диапазону дат текущего view.
12. **Гигантский queryKey `useBoardListTimes.ts:15`** — join 2000 UUID ≈ 74 КБ строка в ключе кэша,
    пересобирается с `.sort()` каждый рендер (`BoardListCalendarView.tsx:102`). Хешировать.
