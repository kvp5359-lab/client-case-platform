# Тестирование после рефакторингов 2026-05-24

**Статус:** ожидает прогон в браузере владельцем.
**Контекст:** за день проведён крупный аудит проекта по 6 осям +
4 волны критических правок + 7 средних волн + мелочи.

Тестировать **после всех правок** — единый прогон по чек-листу
ниже. Если в каком-то сценарии что-то сломано — открыть отдельный
баг в `docs/bugs/open/` с привязкой к коммиту.

## Что прогонять (по разделам приложения)

### 🏠 Главное

- [ ] Войти в проект, переключиться между вкладками TaskPanel:
      «Задачи» / «История» / «Документы» / «Ассистент».
      У «Ассистента» допустим короткий спиннер при первом открытии (lazy Tiptap).
- [ ] Создать тред (задачу) в проекте — должен создаться без ошибок.
- [ ] Открыть существующий тред, изменить статус, дедлайн, исполнителя.
- [ ] Удалить тред, восстановить из корзины.

### 📋 Доски

- [ ] Открыть доску, создать колонку, перетащить задачу между колонками.
- [ ] Создать колонку с режимом «календарь» — должен подгрузиться
      календарь со спиннером, потом отрисоваться.
- [ ] Перетащить задачу на временной слот в календарной колонке.
- [ ] Изменить ширину колонок в настройках доски.

### 📅 Календарная страница

- [ ] Открыть `/workspaces/<id>/calendar`.
- [ ] Переключить вид: День / Неделя / Будни / Список.
- [ ] Перетащить задачу — должна обновиться start_at/end_at.
- [ ] Создать задачу слотом (выделить интервал мышью).

### 📁 Документы

- [ ] Открыть вкладку «Документы» в проекте.
- [ ] Загрузить документ в слот.
- [ ] Скачать документ.
- [ ] Переместить документ между папками.
- [ ] Объединить (merge) несколько документов в один.
- [ ] Удалить документ, восстановить из корзины.

### 📚 База знаний

- [ ] Открыть KB-статью на редактирование — Tiptap должен подгрузиться
      со спиннером, потом редактор открыться.
- [ ] Сохранить изменения статьи.
- [ ] Создать тег статьи.
- [ ] Создать группу статей.
- [ ] Управление группами/тегами — диалоги работают.

### 💬 Быстрые ответы (справочник)

- [ ] Открыть `/workspaces/<id>/directories/quick-replies`.
- [ ] Создать новый быстрый ответ (Tiptap должен подгрузиться).
- [ ] Отредактировать существующий.
- [ ] Сгруппировать ответы по группам.

### 👥 Участники и роли

- [ ] Открыть `AssigneesPopover` в задаче — выбрать исполнителя.
      **Важно:** Владельцы и Исполнители должны быть в группе «Сотрудники»
      (а не «Прочие») — починен баг multi-select.
- [ ] Открыть `ParticipantsPicker` — добавить участника в проект.
- [ ] В настройках воркспейса — заблокировать участника (`can_login=false`).

### 🔍 Глобальный поиск

- [ ] В сайдбаре открыть поиск, ввести запрос — найти проект, тред,
      статью KB, участника.
- [ ] Кликнуть на результат — должен открыться правильный объект.
- [ ] Проверить раздел «Недавнее».

### 📊 Дневник проекта

- [ ] Открыть «Дневник» в проекте — сгенерировать сводку дня.
- [ ] Поменять системный промпт в настройках воркспейса.
- [ ] Запустить пакетную генерацию `/workspaces/<id>/digests`.

### ⚙️ Настройки воркспейса

- [ ] Открыть `/workspaces/<id>/settings` — все вкладки рендерятся.
- [ ] Включить/выключить модуль в шаблоне проекта.
- [ ] Изменить настройки сайдбара (drag-and-drop, папки).

### 📦 Списки (item lists)

- [ ] Открыть `/workspaces/<id>/lists/<id>` — таблица тредов.
- [ ] Изменить фильтр — список перестраивается.
- [ ] Чекбокс + пакетная смена статуса.

### 📥 Импersonация

- [ ] Зайти как Владелец, выбрать другого юзера в `ParticipantMenu` →
      «Войти под этим пользователем».
- [ ] Проверить, что в баннере сверху видно «Режим просмотра».
- [ ] Попробовать сделать любую запись — должен прилететь toast «Read-only».
- [ ] Выйти из режима, проверить, что вернулся обратно как Владелец.

### 📋 Корзина

- [ ] Удалить проект → проверить, что попал в корзину.
- [ ] Восстановить из корзины.
- [ ] Hard-delete из корзины.

- Мессенджер (карантин — не трогал)
- Email / Gmail (карантин)
- MTProto / Telegram Business (карантин)

Эти разделы должны работать как и раньше — менять их не было. Но
быстрый дым-тест («открылось, сообщение отправилось/получилось») — стоит.

## Если что-то сломалось

1. Откатить конкретный коммит (`git revert <sha>`) или вернуться на
   `main` до начала рефакторинга — все коммиты в истории.
2. Завести `docs/bugs/open/2026-05-XX-<краткое-описание>.md`.
3. Прислать мне название коммита, файл, в котором падает, и текст ошибки.
