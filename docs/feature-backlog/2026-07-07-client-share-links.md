## Задача (простыми словами)

1. **Публичные ссылки на статьи БЗ для клиента.** Ссылка открывается у любого без авторизации, но **перестаёт работать, когда проект переходит в финальный статус**. Одна статья в разных проектах = разные ссылки, каждая «протухает» вместе со своим проектом.
2. **Кнопка «Поделиться ссылками» в треде** — собирает все полезные ссылки проекта (статьи + Google Drive/Sheets + документы) в список, выбор галочками → вставка в сообщение клиенту.

## Принятые решения (с владельцем)

- Формат шеринга — **набор отдельных ссылок** (не единая страница проекта).
- Внешние ссылки (Google Drive/Sheets) — **отдаём напрямую**, без нашего шлюза. Закрываем при финале **только наши статьи**.
- Публичная страница статьи — **всегда read_only** (копирование заблокировано), независимо от `access_mode` статьи.
- Управление ссылками — **авто-режим** (живёт, пока проект не в финале) + кнопка «пересоздать». Отдельной панели управления НЕ делаем.

## Часть 1 — Публичные ссылки на статьи

### Таблица `article_share_links`

| Поле | Тип | Смысл |
|---|---|---|
| `id` | uuid PK | |
| `token` | text UNIQUE | Неугадываемый код в URL (~128 бит, base62/nanoid) |
| `article_id` | uuid FK → knowledge_articles | |
| `project_id` | uuid FK → projects | |
| `workspace_id` | uuid | |
| `created_by` | uuid | |
| `created_at` | timestamptz | |
| `revoked_at` | timestamptz null | ручной отзыв / «пересоздать» отзывает старую |

`UNIQUE (article_id, project_id)` где `revoked_at IS NULL` (partial) → одна активная стабильная ссылка на пару. RLS: SELECT/INSERT/UPDATE участникам воркспейса; anon НЕ имеет доступа к таблице напрямую (только через RPC).

### RPC

1. **`get_shared_article(p_token text)`** — `SECURITY DEFINER`, `GRANT anon` (класс безопасности как у существующих short_id-резолверов). Отдаёт `{ title, content }` **только если**:
   - токен есть и `revoked_at IS NULL`;
   - проект НЕ в финале: `COALESCE(pts.is_final, s.is_final, false) = false` (join `projects → statuses` + `project_template_statuses` по `template_id+status_id`);
   - `knowledge_articles.is_deleted = false` (если есть) и `projects.is_deleted = false`.
   - Иначе — NULL / код-ошибка (страница показывает «Ссылка недействительна»).
   - ⚠️ Картинки: если бакет со статейными картинками приватный — переписать относительные Storage-пути на публичные/подписанные URL прямо в RPC (см. риск ниже).

2. **`ensure_article_share_link(p_article_id uuid, p_project_id uuid)`** — authenticated, участник воркспейса. Возвращает существующий активный токен для пары или создаёт новый. Проверяет, что статья реально доступна в проекте (см. определение ниже) — чтобы не шарить произвольное.

3. **`regenerate_article_share_link(p_article_id, p_project_id)`** — отзывает старую (`revoked_at = now()`) + создаёт новую. (Кнопка «пересоздать».)

### Публичный роут `/a/<token>`

- Новый сегмент вне auth-зоны. Страница: fetch `get_shared_article` (anon-клиент) → рендер HTML через `sanitizeHtml`, read_only (блок выделения/копирования как в `KnowledgeArticleTabContent` при `access_mode='read_only'`).
- **`src/proxy.ts`**: добавить `/a/*` в белый список публичных путей (сейчас неавторизованных редиректит на `/login`). 🟠 Чувствительный файл — правка точечная.
- Токен глобально уникален → путь host-agnostic; ссылку генерируем на основном хосте воркспейса (slug/custom_domain) ради брендинга.

## Часть 2 — Сборщик ссылок в треде

### RPC `get_project_shareable_resources(p_project_id uuid)`

Собирает и возвращает список ресурсов проекта:

**Статьи, доступные в проекте** (union, distinct):
- через шаблон: `knowledge_article_templates` где `project_template_id = projects.template_id`;
- через группы шаблона: `knowledge_group_templates → knowledge_article_groups`;
- через папки/слоты: `folders.knowledge_article_id` / `folder_slots.knowledge_article_id` где `project_id = p_project_id`.

**Внешние ссылки:**
- `projects.google_drive_folder_link` (папка Drive);
- `form_kits.google_sheet_id` (анкета) и `brief_sheet_id` (бриф) → `https://docs.google.com/spreadsheets/d/<id>/edit`;
- `source_documents.web_view_link` (документы Drive).

### UI

- Кнопка «Поделиться ссылками» в меню «⋮» треда (`TaskActionsMenu.tsx`, после «Настроек»). Виден только если `thread.project_id != null`.
- Диалог: список ресурсов с чекбоксами. У статей — кнопка получить/пересоздать ссылку (`ensure_article_share_link` / `regenerate_...`), строим публичный URL. Внешние — как есть.
- Кнопка «Вставить в сообщение» → через существующий `insertContentRef` (`MessengerTabContent` → `MessageInput`) кладёт выбранное HTML-списком в композер. Плюс индивидуальное «копировать».
