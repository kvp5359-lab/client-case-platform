# Полноценная админка платформы (/admin)

Дата плана: 2026-07-04. Фундамент уже есть: `platform_admins`, `plans`/`workspace_billing`,
`ai_usage_*`, `platform_alert_config` + крон алертов, страница `/admin` (список воркспейсов + смена тарифа).

## Блоки

1. **Воркспейсы** — статус (активен/триал/заблокирован), последняя активность, блокировка целиком
   (`workspaces.is_suspended` + server-гард в layout), карточка воркспейса (владелец, участники,
   интеграции, потребление).
2. **Пользователи** — все аккаунты: email, последний вход, воркспейсы/роли; бан аккаунта
   (UPDATE auth.users.banned_until + revoke сессий), сброс пароля.
3. **Тарифы и биллинг (шаг 1, без платёжки)** — редактор тарифов в UI; поля «оплачено до»/«триал до»;
   ручная отметка оплаты + история (`platform_payments`); статусы trial → active → past_due;
   крон просрочки + алерт в TG. Шаг 2 (Stripe/ЮKassa) — отдельный проект.
4. **Потребление** — топы по токенам/хранилищу/сообщениям, динамика по месяцам, алерт >90% лимита.
5. **Вход под клиента** — расширение импersonации: платформенный админ → владелец любого воркспейса.
   Read-only, TTL 30 мин, обязательный аудит. Отдельный коммит/сессия (JWT).
6. **Здоровье платформы** — UI-зеркало `scripts/channel-health.mjs`: застрявшие отправки,
   незакрытые сбои, протухший Gmail watch, падения кронов — с привязкой к воркспейсам.
7. **Метрики роста** — регистрации по неделям, активные воркспейсы 7/30д, распределение тарифов, выручка.
8. **Аудит действий админа** — `platform_admin_audit`, запись в каждом админском RPC, вкладка «Журнал».
9. **Объявления** — `platform_announcements` → баннер в сервисе; выгрузка email владельцев.
10. **Регистрация** — `platform_config` (открыта/по инвайтам, дефолтный триал), инвайт-ссылки.

## Принципы

- Все админ-RPC: `SECURITY DEFINER` + первая строка `require_platform_admin()`; REVOKE anon.
- Каждое изменяющее действие пишет в `platform_admin_audit` (сервер, не фронт).
- Миграции аддитивные, применяются через MCP + файл в репо (полный 14-значный таймстемп!).
- Фронт зовёт RPC через строковый `rpc(... as never)` — админ-функции не генерятся в database.ts.
- Никаких новых npm-зависимостей (графики — CSS-бары).
