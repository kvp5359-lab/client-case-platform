# @-упоминания: список сужен до людей, связанных с задачей

**Дата:** 2026-07-02
**Тип:** fix (UI)
**Статус:** completed (ждёт деплоя фронта)

---

## Проблема

В попапе @-упоминаний выводилось слишком много лишних людей: для проектной
задачи — **все** участники проекта, для задачи без проекта — **все** сотрудники
воркспейса.

## Что сделано

Список кандидатов теперь = **объединение трёх источников по этой задаче**:

1. участники проекта (`useProjectParticipants`),
2. участники задачи / thread members (`useThreadMembers`),
3. исполнители (`useTaskAssigneeIds`).

Больше никого. Из справочника воркспейса (`useWorkspaceParticipants` — источник
имён/аватаров/`user_id`) берутся только те, чей id попал в одно из трёх множеств,
**с аккаунтом** (`user_id` — telegram-контакты отсеиваются) и не текущий
пользователь. Ветка «orphan → весь воркспейс» убрана.

## Решение по клиентам

По согласованию — «все три источника как есть»: клиент-участник проекта **с
аккаунтом** в списке остаётся (он законный участник). Отфильтрованы только
несвязанные с задачей сотрудники воркспейса и контакты без аккаунта.

## Нагрузка

Плюс два лёгких per-thread запроса (`participant_id` из `project_thread_members`
и `task_assignees`) — кэшируются, обычно уже загружены для треда. Справочник
воркспейса — один кэш-запрос на воркспейс, общий по приложению.

## Грабли

- Пул упоминаний = union(проект-участники ∪ thread members ∪ исполнители),
  резолв деталей — через `useWorkspaceParticipants` (все три — участники
  воркспейса). Новый источник «кого можно упомянуть» держать здесь.
