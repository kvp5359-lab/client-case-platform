# Доски: автосворачивание пустых списков

**Дата:** 2026-04-15
**Тип:** feat
**Статус:** completed

---

## Проблема

При открытии доски все списки разворачивались по умолчанию, даже пустые. Глаз цеплялся за «нулевые» колонки, экран засорялся.

## Решение

В [`BoardListCard.tsx`](../../src/components/boards/BoardListCard.tsx) состояние свёрнутости стало производным:

```ts
const [userCollapsed, setUserCollapsed] = useState<boolean | null>(null)
const collapsed = userCollapsed ?? (!isInbox && count === 0)
```

- Если пользователь ещё не трогал список — показываем свёрнутым при `count === 0` (после фильтров).
- Как только данные подгружаются и `count > 0` — список сам разворачивается.
- После ручного клика по заголовку фиксируем выбор пользователя в `userCollapsed` и игнорируем автологику.
- Inbox исключён — он всегда развёрнут по умолчанию.
