# CI: усилены ретраи SSH-деплоя (транзиентный обрыв Actions→VPS)

**Дата:** 2026-07-02
**Тип:** ci / infra
**Статус:** completed

---

## Проблема

Деплой дважды подряд падал на шаге «Deploy to VPS via SSH» с
`ssh: connect to host … port 22: Connection timed out` (все 3 попытки).
Диагностика показала: **VPS исправен** — `ufw` inactive, `fail2ban` не
установлен, во INPUT nftables блокировки 22 нет, sshd слушает; с локальной
машины `ssh` проходит, ручной re-run job'а проходит. Значит обрыв — на **сетевом
пути между GitHub-раннером и VPS** (потеря пакетов на ~5 мин, затем
самовосстановление). Не код, не хост.

Прежние ретраи (3 попытки × ConnectTimeout 60с + пауза 20с ≈ 4 мин) укладывались
внутрь окна провала → все три падали.

## Что сделано

`.github/workflows/deploy.yml`, шаг retry:
- `max_attempts` 3 → **6**
- `retry_wait_seconds` 20 → **45**
- `ConnectTimeout` 60 → **30** (быстрее отбраковывать мёртвую попытку)
- `timeout_minutes` 12 → **20** (запас)
- добавлены `-o ServerAliveInterval=15 -o ServerAliveCountMax=4` (держит уже
  установленную сессию — deploy-скрипт идёт ~2 мин)

Итог: попытки растянуты на ≈7.5 мин — перекрывают наблюдаемое окно провала,
ловят момент восстановления пути (как это делает ручной re-run).

## Если не поможет (эскалация)

- self-hosted runner прямо на VPS — SSH-путь исчезает совсем;
- pull-деплой — VPS сам тянет образ по webhook, без входящего SSH от Actions.

`scripts/deploy-vps.sh` идемпотентен → повторные попытки безопасны.

**Файлы:** `.github/workflows/deploy.yml`.
