## 1. Быстрое добавление (QuickAddModal)

Новый компонент [`QuickAddModal`](../../src/components/plan/QuickAddModal.tsx):

- При наведении на строку **задачи или заголовка** в плане под ней по центру
  появляется кнопка «+» ([`ProjectFlatPlanList`](../../src/components/plan/ProjectFlatPlanList.tsx) → `QuickAddBelow`). Клик открывает модалку.
- **Сверху** — селектор типа (переключатель одного типа): Задача / Заголовок / Текст.
- **Поле ввода**: для «Текст» — `textarea` (Enter добавляет в список, Shift+Enter — перенос строки), для остального — однострочный `input`.
- **Список к добавлению**: каждый элемент с тегом типа (кликабельный — меняет тип через дропдаун), DnD-перетаскивание порядка, удаление.
- **«Добавить (N)»** создаёт все по очереди и вставляет в позицию, где нажали «+»: существующие элементы после точки сдвигаются на `N*10`, новые занимают окно. Блоки создаются сразу с нужным `sort_order`, задачам он проставляется после создания (`onReorderTasks`).
- Текст/заголовок сохраняются как HTML-параграфы (`<p>` на строку) — переносы корректно отображаются (`htmlToPlain` + `whitespace-pre-wrap`) и правятся в Tiptap.
- Состояние сбрасывается при каждом открытии (тело внутри `DialogContent`, Radix размонтирует).

## 2. Видимость типов блоков

- Тумблеры «Заголовки / Текст / Документы» перенесены из футера плана в **сворачиваемую панель фильтров** ([`TaskListControls`](../../src/components/tasks/TaskListControls.tsx)), показываются только в plan-режиме.
- Состояние — общий хук [`usePlanBlockVisibility`](../../src/hooks/plan/usePlanBlockVisibility.ts) (поднят в `TaskListView`), запоминается в `localStorage` (`cc:plan-block-visibility-v1`).

## 3. Полировка плана

- Текстовые блоки по умолчанию **свёрнуты** в одну строку.
- Шеврон сворачивания выровнен по центру первой строки текста.
- Под текстом убран горизонтальный разделитель.
- Убраны футер-кнопки «Заголовок/Текст/Документ» и финальный разделитель под последней строкой списка.
- Фикс hydration-ошибки: вложенный `<button>` в SlotPicker → `<div role="button">`.

## 4. UX строк задач

- Срок прижат к аватарам исполнителей (раньше улетал к правому краю) — [`TaskRow`](../../src/components/tasks/TaskRow.tsx). Заполненный виден всегда, пустой «Срок» — только на hover.
- Цветовые акценты срока в [`DeadlinePopover`](../../src/components/tasks/DeadlinePopover.tsx): просрочено → красный, сегодня → оранжевый, завтра → синий, послезавтра → зелёный, позже → серый. По разнице дней, без расширения общего `DeadlineGroup`.
- Активная (открытая в панели) задача подсвечивается светлее (`bg-gray-100`).
- Фикс пикера дат [`TaskTimePickerPopover`](../../src/components/tasks/TaskTimePickerPopover.tsx): перемотка месяца с 31-го числа больше не пропускает короткий месяц (привязка к 1-му числу вместо `setMonth`).
