# Доступ клиентов по паролю + доработка списка участников

**Дата:** 2026-06-21
**Тип:** feat (новая фича + UX)
**Статус:** completed (Edge-функция в проде, фронт ждёт деплоя CI)

---

Две связанные доработки вокруг участников: (1) выдача клиентам доступа в
личный кабинет по логину/паролю вместо обязательного Google OAuth;
(2) переработка списка участников в настройках воркспейса.

## 1. Доступ клиента по паролю

Раньше у клиента-участника не было аккаунта для входа — он появлялся, только
если клиент сам регистрировался. Теперь менеджер выдаёт доступ из карточки.

- **Edge Function `set-client-password`** (`verify_jwt`, права = владелец или
  `manage_workspace_settings`): генерит читаемый пароль (без неоднозначных
  символов), создаёт клиенту auth-аккаунт (или привязывает существующий, если
  email уже занят), ставит `user_id` + `can_login=true`. Пароль возвращается
  один раз для показа менеджеру; в БД — только хеш. Логин = email.
- **Выдача из UI** — две точки входа: круглая кнопка-ключ рядом со статусом
  «Доступ» в карточке участника (`EditParticipantDialog` → `ClientAccessDialog`)
  и пункт меню «Выдать доступ по паролю / Сбросить пароль» в списке
  (`ParticipantMenu` → `ClientPasswordDialog`). Окно показывает логин+пароль
  с копированием готового сообщения.
- **Отзыв доступа** — переиспользует существующую блокировку
  (`set-participant-access`: `can_login=false` + сброс сессий).
- **Вход по паролю** на `/login` стал явной кнопкой «Войти по логину и паролю»
  (был спрятан как legacy-ссылка).
- **Смена пароля клиентом** — секция «Сменить пароль» в профиле
  (`ChangePasswordSection`, `supabase.auth.updateUser`).
- **Копирование с fallback** (`utils/clipboard.ts`) — `navigator.clipboard`,
  при недоступности (небезопасный контекст / iframe) → `execCommand` через
  скрытый textarea.

Миграций БД не понадобилось — состояние «доступ выдан» выводится из
`participants.user_id`/`can_login`.

## 2. Список участников (настройки → Участники)

- **Поиск** по всем полям (имя, фамилия, email, телефон, Telegram, роль).
- **Автоподгрузка при скролле** (порции по 50, IntersectionObserver) вместо
  кнопочной пагинации.
- **Фиксированная левая панель** ролей — контейнер фикс-высоты
  (`h-[calc(100vh-13rem)]`), скроллится только список.
- **Адаптивная таблица** (`table-fixed`) — колонка «Доступ + меню» больше не
  уезжает; имя/email обрезаются многоточием; `pr-3` чтобы overlay-скроллбар
  (macOS) не накрывал меню.
- **Высота строки −25%** (паддинги + аватар 24px).
- **Бейдж роли** без переноса; колонка «Роль» расширена под «Внешний контакт».
- **Телефон → колонка «Контакты»** с иконками каналов (телефон/Telegram) и
  тултипом по наведению.

## 3. Карточка контакта / пикер участников

- **Пикер** (`ParticipantsPicker`, общий для проекта и задач): выбранные —
  отдельной секцией наверху; крестик удаления на чипе (как в треде); иконка
  открытия карточки при наведении на строку.
- **Полная карточка** участника открывается сразу в режиме редактирования
  через стор (`contactCardStore.fullEdit`, `key` на диалоге).
