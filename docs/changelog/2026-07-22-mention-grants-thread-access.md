# @-упоминание выдаёт доступ к треду + все сотрудники в списке

**Дата:** 2026-07-22
**Тип:** feature
**Статус:** completed (БД в проде; фронт — этим деплоем; смок за владельцем)

---

## Контекст

Сотрудника, не состоящего в проекте, нельзя было отметить через `@` в треде проекта — список кандидатов ограничивался людьми, связанными с задачей (участники проекта ∪ участники задачи ∪ исполнители). Владелец выбрал вариант: показывать в списке ВСЕХ сотрудников воркспейса, а упоминание человека без доступа — автоматически добавлять его участником задачи. Иначе получалось «отметили в пустоту»: подписка от упоминания без доступа ничего не показывает (RLS не пускает, тред не виден во «Входящих»).

## Что сделано

### БД (прод через MCP + миграция)

- [`supabase/migrations/20260722190000_mention_grants_thread_access.sql`](../../supabase/migrations/20260722190000_mention_grants_thread_access.sql) — триггер `trg_mention_recompute` расширен: упомянутый **сотрудник** (workspace-роль из `is_staff_role`, есть аккаунт, `can_login`) без доступа к треду добавляется в `project_thread_members` → получает доступ через member-ветку `can_user_access_thread`.
- **Отмотка baseline:** `seed_read_status_on_thread_member` сидирует точку прочтения «сейчас» (позже упоминающего сообщения — оно оказалось бы «прочитанным»). После выдачи доступа baseline отматывается на 1 мкс раньше сообщения: непрочитанным становится ровно упоминающее сообщение, старая история треда остаётся прочитанной.
- **Гейты (по итогам ревью):** доступ выдаётся только в тредах ПРОЕКТА (в личных диалогах упоминание раздавало бы коллегам доступ к личной переписке); не для сообщений «Только я» (`visibility='self'` скрыто RLS — была бы тихая выдача доступа без сигнала); клиентам/без аккаунта — никогда. Провал обработки логируется `RAISE WARNING` (не глотается молча).

Связанные первыми, обе группы по алфавиту.
- [`src/components/messenger/hooks/useMentionItems.ts`](../../src/components/messenger/hooks/useMentionItems.ts) — тонкая обёртка над `buildMentionItems`.

- [`src/hooks/messenger/useSendMessage.ts`](../../src/hooks/messenger/useSendMessage.ts) — после отправки с упоминаниями инвалидируется кэш участников треда («Кто видит задачу» обновляется без перезагрузки).

## Миграции / Edge Functions

Миграция применена в прод через MCP (2 итерации: базовая фича + пост-ревью гейты). Edge Functions не менялись. ⚠️ Дрейф манифеста: `trg_mention_recompute` — обновить `db-drift-check --update` при случае.

## Известные ограничения

- Пути «Запланировать» / «Сохранить черновик» упоминания не извлекают (пред-существующий пробел): тег в отложенном сообщении виден, но доступ/подписка не выдаются.
- В личных диалогах общий список сотрудников намеренно скрыт — упоминание там работает по-старому (только связанные).
