# Доски: доступ «Выбранные» (участники) + «Название по умолчанию» у типа проекта

**Дата:** 2026-06-15
**Тип:** feat + fix
**Статус:** completed

Две доработки CRM-контура: рабочий выбор участников для приватной доски и
префикс имени у типа проекта (лида).

---

## 1. Доступ к доске «Выбранные» — выбор участников

**Было:** в диалогах создания/настройки доски режим «Выбранные» (`access_type='custom'`)
можно было выбрать, но **не было UI выбора людей** — доступ нельзя задать, доска
оставалась видна только создателю. Опция фактически нерабочая.

**Корень:** `CreateBoardDialog`/`EditBoardDialog` писали `access_type='custom'`
с пустым набором; read-side (`get_workspace_boards`) уже пускал по
`board_members` и `access_roles`, но писать их было нечем.

**Решение:**
- При «Выбранные» показываем `ParticipantsPicker` (поиск, группировка по ролям) —
  тот же, что в тредах/проектах.
- Выбор пишется в `board_members` (RLS уже разрешает участнику воркспейса).
- `useCreateBoard`/`useUpdateBoard` принимают `memberIds` и синкают `board_members`
  (create — insert; update — delete + insert). Хук `useBoardMembers(boardId)`
  подгружает существующих для редактирования.
- ⚠️ **Footgun закрыт:** для `custom` доступ проверяется только по ролям/участникам
  (НЕ по `created_by`) — поэтому создатель/редактор всегда добавляется в набор,
  иначе потерял бы доступ к собственной доске.

## 2. «Название по умолчанию» у типа проекта

Поле-префикс имени для проектов этого типа (CRM: лид «Лид: …»).

- **БД:** `project_templates.default_name_prefix text` (миграция
  `20260613_project_template_default_name_prefix.sql`).
- **Редактор шаблона:** поле «Название по умолчанию» под флагом «Это шаблон лида»;
  сохраняется на blur.
- **Создание проекта:** при выборе типа в `CreateProjectDialog` префикс
  подставляется **в начало** имени (имя контакта `Кирилл` + `Лид:` → `Лид: Кирилл`).
  Дальше имя редактируется свободно; при смене типа старый префикс заменяется
  новым (не накапливается).

**Fix кэша:** мутация префикса инвалидирует и **список** шаблонов (broad-ключ
`['project-templates']`), а не только `detail`. Иначе при глобальном
`staleTime: LONG` диалог «Создать проект» читал список из кэша со старым
префиксом и подстановка «не срабатывала».
