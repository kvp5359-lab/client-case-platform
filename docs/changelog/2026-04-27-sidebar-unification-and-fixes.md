# Унификация настроек сайдбара + два фикса

**Дата:** 2026-04-27
**Тип:** feature + bugfix
**Статус:** completed

---

## Контекст

Три самостоятельных проблемы, попавших в одну поставку:

**Настройки сайдбара выглядели лоскутно.** Два независимых блока на одной странице с разной логикой: «Пункты меню» — селектор `topbar/list/hidden`, отдельная секция «Скрытые». Ниже — «Доски в сайдбаре» с собственным селектором закрепления и собственной сортировкой, своей разметкой и своим набором бейджей. Пользователь сказал: «всё запутано — должно быть две зоны (верх и список) плюс пул доступных, а перенос — кнопками. Доски и пункты — однотипно».

**Бейдж в Telegram «Кирилл:» появлялся слишком часто.** Раньше окно группировки сообщений одного автора было 5 минут — если сотрудник писал в 10:00, потом в 10:06 без вклинивания других участников, имя появлялось заново. Это читалось как «новый человек в чате».

**Карточка задачи на доске оставалась с обводкой после скрытия панели.** Открыл задачу через клик на карточке → панель появилась, карточка получила outline. Нажал «Скрыть панель» (оставляет вкладки) — панель ушла, обводка осталась.

## Изменения

### Унификация настроек сайдбара (миграции 20260427_workspace_sidebar_unified_slots.sql)

**Дата-модель.** Единая колонка `slots` (jsonb) в `workspace_sidebar_settings` вместо двух (`items` + `pinned_boards`). Каждый слот:

```json
{
  "id": "nav:home" | "board:<uuid>",
  "type": "nav" | "board",
  "placement": "topbar" | "list",
  "order": 0,
  "badge_mode": "disabled" | "my_active_tasks" | "all_my_tasks" | "overdue_tasks" | "unread_messages" | "unread_threads"
}
```

«Скрытые» больше не хранятся в БД — это просто отсутствие записи в `slots`. Пункт меню и доска теперь однотипны, вся разница между ними — в `type` и в том, как резолвится подпись/иконка.

Миграция применяется в три шага (предыдущие два были untracked-черновиками этой же фичи): `workspace_sidebar_settings.sql` → `workspace_sidebar_pinned_boards.sql` → `workspace_sidebar_unified_slots.sql`. Финальная конвертирует `items` + `pinned_boards` в `slots` и дропает старые колонки.

**UI настроек — три зоны.** Страница `/workspaces/<id>/settings/sidebar`:

1. **Верхняя строка** — иконки в шапке сайдбара, размещённые слоты с `placement: 'topbar'`.
2. **Список** — полные пункты в основном списке, `placement: 'list'`.
3. **Доступные** — всё, чего нет в `slots`: оставшиеся пункты меню + все доски воркспейса.

Перенос между зонами — кнопками: `+ в верх` / `+ в список` (из «Доступных»), стрелка на другую зону (внутри размещённой), `×` (вернуть в «Доступные»). Порядок внутри зоны — стрелками ↑↓. Drag-n-drop сознательно не делал — пользователь явно попросил «пока стрелки, не усложняем».

**Бейджи стали универсальны.** Раньше у пунктов меню были захардкоженные бейджи (Inbox → непрочитанные треды, Tasks → активные задачи), у досок — настраиваемые. Теперь у каждого размещённого слота свой селектор `badge_mode` из единого набора. Можно поставить «непрочитанные треды» хоть на пункт «Главная», хоть на доску. Счётчики глобальные по воркспейсу — содержимое конкретной доски не учитывается.

«Мои активные задачи» переименован в «Задачи на сегодня» — точнее отражает суть (на сегодня + просрочка), пользователь сам предложил.

**Закрепление досок переехало в slots.** Старая per-user таблица `pinned_boards` уже не использовалась — это уровень воркспейса. Хук `usePinnedBoards` стал тонким адаптером над `slots`: `togglePin` добавляет/удаляет board-слот в `placement: 'list'`. `BoardsPage` ничего не заметил.

**Мёртвые слоты** (доски, удалённые из воркспейса) фильтруются на рендере. На странице настроек владельцу показывается баннер «есть мёртвые элементы — очистить?».

### Telegram: окно группировки 5 → 60 минут

В edge function `telegram-send-message`: переменная `fiveMinAgo` → `sixtyMinAgo`. Логика осталась прежней — берём последнее сообщение в этом канале за окно, и если автор тот же и `source = 'web'`, имя скрываем. 60 минут лучше совпадает с темпом обычной переписки и не сбрасывается на короткой паузе.

Edge function задеплоена через `supabase functions deploy --no-verify-jwt` отдельным шагом — workflow деплоит только Next-приложение, функции едут вручную.

### Карточка на доске не сбрасывала обводку после `hidePanel`

В `TaskPanelTabbedShell.tsx`: `activeThreadId` и `activeProjectRefId` (значения, по которым `BoardTaskRow.isSelected` ставит outline) теперь возвращают `null` при `hidden = true`. Раньше они смотрели только на `tabs.activeTab` — а вкладка остаётся активной, когда панель просто скрыта (это и задумывалось — вернуть «как было» по `togglePanel`). Outline должен идти за видимостью, а не за наличием активной вкладки.

## Почему так

**Унификация vs два независимых блока.** Раньше казалось, что «пункты меню» и «доски» — разные сущности с разной семантикой. На практике — одна и та же абстракция: «то, что показывается в верхней части сайдбара, в одной из двух зон, с опциональным счётчиком». Разделять их — раздувать UI и код. После объединения страница настроек сократилась до трёх однотипных карточек, у каждого слота одинаковый набор контролов.

**Окно 60 мин против 5 мин.** В идеале правильнее было бы вообще отвязаться от таймера и группировать по «подряд идущим сообщениям одного автора» — но пользователь хотел оставить таймер для случая «после долгой паузы в чате имя должно появиться снова, даже если никто не вклинился». 60 мин — компромисс: для обычной переписки имя не появляется лишний раз, после реального перерыва — появляется.

**Outline и hidden.** «Скрыть панель» (а не «закрыть») — намеренная фича: вкладки сохраняются, можно вернуться. Но визуальная связь карточки с панелью должна жить ровно столько, сколько панель видна. Иначе пользователь видит выделенную карточку и думает «а где же открытая задача?».
