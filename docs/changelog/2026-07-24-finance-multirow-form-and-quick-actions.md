## 1. Многострочная форма ввода дохода/расхода (feature)

Диалог «Новый доход/расход» переработан под пакетный ввод: раньше — одна
операция на открытие, теперь проект выбирается один раз сверху, а операции
вносятся строками таблицы.

- Диалог стал широким (`sm:max-w-3xl`); каждая запись — две строки:
  «Дата · Статья · Налог · Сумма» и «Кому/От кого · Комментарий».
- «+ Ещё строка» добавляет запись (дата наследуется от предыдущей, налог —
  дефолтный из справочника), крестик убирает лишнюю. Кнопка показывает
  счётчик: «Добавить (N)».
- Все записи создаются **одним запросом** в БД (батч-insert, атомарно) —
  `useCreateWorkspaceTransaction` / `useCreateProjectTransaction` теперь
  принимают массив форм.
- Пустая строка-заготовка молча игнорируется; строка с данными без корректной
  суммы блокирует сохранение — ничего не теряется.
- Редактирование существующей операции — та же вёрстка, но одна запись без
  добавления строк.
- Клик по любому месту поля даты открывает нативный календарь (`showPicker`) —
  раньше Chrome ждал клика точно по иконке-индикатору.
- Работает в обоих местах: общий журнал «Финансы» воркспейса и вкладка
  «Финансы» проекта (там сохранён тег «Остаток» — клик подставляет сумму в
  первую запись без суммы).

Обвязка создания для общего журнала (проекты, валюта, мутация, тосты)
вынесена в самодостаточный `WorkspaceTransactionCreateDialog` — его используют
и страница «Финансы», и быстрое действие «+» (см. ниже). Чистая логика строк —
в `src/lib/finance/transactionEntryRow.ts` (+6 тестов); общая валидация пачки —
`saveTransactionForms.ts`.

## 2. Пикер проектов из треда — в форме операций (UX)

Поле «Проект» в диалоге операций — тот же `ChatSettingsProjectSelector`, что в
шапке треда: единый список с иконками/цветами проектов как в сайдбаре, поиск и
«+» для создания проекта из шаблона прямо из формы. «Без проекта» снимает
выбор — операция без проекта не сохраняется (кнопка гаснет).

Попутно починен сам пикер (влияет и на шапку треда): при длинных именах
проектов список получал горизонтальный скролл — теперь `overflow-x-hidden` +
имя с многоточием. Из `useChatSettingsData` экспортирован единый тип
`WorkspaceProjectOption`. Детально — `messenger-ledger.md`, запись 2026-07-24.

## 3. Быстрое действие «Доход / расход» в меню «+» (feature)

Новый kind `new_transaction` у быстрых действий (`interface_presets.config.
quick_actions`) — по образцу `new_thread` + параметр:

- в редакторе (Настройки → Сайдбар → «Быстрые действия „+"») — тип
  «Доход / расход» с селектом типа операции; админ создаёт две кнопки со
  своими названиями/иконками (дефолт — кошелёк);
- клик открывает ту же многострочную форму (лениво — финансовый код не попадает
  в бандл сайдбара);
- 🔒 кнопки скрыты у клиентов и у всех без права `view_finance` — тот же гейт,
  что у пункта сайдбара «Финансы» (`QuickActionSlotButton`, работает и для
  кнопки-слота, и внутри папки «+»). Гейт продуктовый (UI): RLS
  `project_transactions` шире by design.
