Основная тема — довести «Группы задач» (`project_task_groups`, задачи входят через
`project_threads.task_group_id`) до удобного вида и заменить в одном боевом проекте
старые «разделители-заголовки» (блоки плана `block_type='heading'`) на группы.
Плюс две попутные правки, накопившиеся в рабочей копии.

## 1. Перегруппировка проекта «CP: Aleksei Nikolaev» (данные, в проде)

В проекте `5cc0f8bb-d748-4454-bae6-34a44032352f` три заголовка-разделителя заменены
на группы задач (через MCP, атомарным запросом; данные, не схема):

| Группа (бывший заголовок) | Задач |
|---|---|
| Разработка бизнес-плана | 5 |
| На стороне клиента | 6 |
| На нашей стороне | 6 |

Каждой группе задан `sort_order` заменяемого заголовка (порядок в списке сохранён
1:1), задачам проставлен `task_group_id`, `visible_to_client=false` унаследован с
заголовков, сами heading-блоки удалены. Блок «СТРАТЕГИЯ:» и всё вне групп оставлены
как есть. Механизм рендера (`usePlanGroupsDnd`): группа встаёт в общий поток по
своему `sort_order`, задачи сортируются внутри по своему — поэтому порядок не поехал.

## 2. UI заголовка группы (`PlanGroupContainer.tsx`)

- **Фон полоски заголовка темнее** — `bg-muted/60` → `bg-muted` (контраст с телом
  группы `bg-muted/20`).
- **Отступ перед группой** — на контейнер добавлен `mt-4` (визуальный зазор сверху,
  именно перед группой).
- **Клик по названию сворачивает/разворачивает** группу (раньше клик открывал
  переименование). Доступно всем, не только редакторам.
- **Карандаш редактирования** — новая кнопка ✏️ сразу справа от текста названия,
  серая (`text-muted-foreground/50`), появляется только при наведении на СВОЮ группу
  (`opacity-0 group-hover/plangroup:opacity-100`), подсветка при наведении на неё.
  По клику — режим переименования (как было по клику на название).
- **Кнопка «+»** — теперь слева от индикатора «скрыта от клиента»; появляется по
  наведению на СВОЮ группу (было `group-hover/planroot` → показывалось во всех
  группах разом; стало `group-hover/plangroup`). Реализована через display-toggle
  (`md:hidden md:group-hover/plangroup:inline-flex`), а не `opacity` — в покое места
  не занимает (убрана пустота между счётчиком и «⋯»). На тач-экране видна всегда.

## 3. «+» группы открывает окно создания (`ProjectFlatPlanList.tsx` + `TaskListView.tsx`)

Раньше «+» в заголовке группы молча создавал задачу с именем «Новая задача» —
неинтуитивно. Теперь открывается то же полноценное окно, что и кнопка «Создать»
(`ChatSettingsDialog`: тип/название/описание/срок/исполнители/первое сообщение/
доступ), а созданная задача автоматически привязывается к этой группе и встаёт в
её конец.

- `ProjectFlatPlanList` считает позицию конца группы (там доступны `blocks` +
  `threadGroupMap`) и прокидывает наверх новый проп `onRequestCreateInGroup(groupId, sortOrder)`.
- `TaskListView` запоминает цель в `pendingGroupRef`, открывает общий диалог, а
  после появления `createdThread` (эффект) привязывает задачу к группе
  (`assignThreadToGroup`) и проставляет `sort_order`. При закрытии диалога без
  создания `pendingGroupRef` сбрасывается (обычное «Создать» не уедет в группу).
- Прежний вариант через `QuickAddModal` откачен — обычное быстрое добавление под
  строкой не изменилось.

## 4. Мгновенное сворачивание группы (`useProjectTaskGroups.ts`)

Сворачивание/разворачивание работало с задержкой ~1с: `updateGroup` (через неё идёт
`setGroupCollapsed`) писала в БД и обновляла UI только после ответа сервера +
рефетча. Добавлено **оптимистичное обновление** (`onMutate` патчит кэш
`taskGroupKeys.byProject` сразу, `onError` откатывает, `onSettled` рефетчит фоном).
Побочно ускорились переименование, смена цвета и переключение видимости группы —
они тоже идут через `updateGroup`.

## 5. Попутные правки

- **Цвет плашки названия проекта** (`InboxChatItem.tsx`, `MessageToastContent.ts`) —
  во «Входящих» и в тостах уведомлений `bg-[#eceef1]` (≈ slate-150) → `bg-slate-200`
  (на шаг темнее, тот же прохладно-серый тон).
- **Смена начального времени сохраняет длительность события**
  (`TaskTimePickerPopover.tsx`) — если у события задан конец и это один день, при
  смене начала конец сдвигается на ту же разницу (12:00–12:30 + начало 14:00 →
  14:30). При многодневном диапазоне (задан `endDate`) конец на другом дне не
  трогается.
