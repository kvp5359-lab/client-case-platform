# 2026-07-21 — Синхронизация слотов набора с шаблоном + фикс потери статьи «Из шаблона»

## Что было не так

- «Обновить состав набора» синхронизировало только **папки**, а слоты существующих
  папок не трогало: переименование слота в шаблоне и добавление статьи не долетали
  до уже созданных проектов.
- Глубже — баг в стыке **шаблон папки → шаблон набора**: RPC `add_folders_to_kit_template`
  при копировании `folder_template_slots → document_kit_template_folder_slots`
  переносила только `name` и `sort_order`, выбрасывая `knowledge_article_id`,
  `slot_template_id`, `description` и AI-промпты. Из-за этого статья не попадала в
  шаблон набора вовсе, и переносить дальше было нечего.

## Синхронизация слотов набор → проект

- Новая колонка-якорь `folder_slots.kit_template_folder_slot_id` (FK →
  `document_kit_template_folder_slots`, `ON DELETE SET NULL`) — стабильная связь
  слота проекта со слотом шаблона. По имени сопоставлять нельзя (имя как раз
  меняется), поэтому нужен id-якорь.
- Якорь заполняется при создании слотов из шаблона: в RPC
  `create_document_kit_from_template` (+ там же теперь протягивается `slot_template_id`)
  и в `buildSlotsFromKitTemplate` (новые папки).
- Бэкафилл существующих связей — по имени слота внутри папки (пока не переименовали,
  имена совпадают), tie-break по `sort_order`. Слоты без совпадения по имени —
  ручные, остаются без якоря.
- `syncDocumentKitStructure` → `syncSlotsForExistingFolders`: для существующих папок
  слот со связью **обновляется** (имя, описание, статья, AI-промпты, `slot_template_id`,
  порядок), слот шаблона без пары **добавляется**. Ручные слоты (без якоря) и
  «осиротевшие» (слот шаблона удалён → якорь занулён FK) **не трогаются и не удаляются**.

## Фикс стыка шаблон папки → шаблон набора

- RPC `add_folders_to_kit_template` теперь переносит в слоты набора все поля
  (`knowledge_article_id`, `slot_template_id`, `description`, AI-промпты) —
  «Из шаблона» больше не теряет статью и связь.
- Бэкафилл существующих шаблонов наборов: дозаполняет **только пустые** поля слотов
  (`COALESCE`, не перетирает заданное вручную) из соответствующих `folder_template_slots`
  по `folder_template` + имени слота. Это **глобальная** одноразовая data-миграция
  (все воркспейсы); `COALESCE`-only-null не отличает «намеренно пусто» от «потеряно
  багом» — осознанный компромисс.

## Поведение (важно знать)

- **Связанные слоты следуют шаблону**: локальная правка имени/описания/статьи у
  *связанного* слота в проекте будет перетёрта шаблоном при следующем «Обновить состав
  набора». Это то же поведение, что уже было у папок (шаг 5 sync). Ручные слоты не
  затрагиваются.
- Новые проекты из шаблона теперь сразу создаются со статьями и связью со справочником
  (без ручного «Обновить»).

## Чистота
