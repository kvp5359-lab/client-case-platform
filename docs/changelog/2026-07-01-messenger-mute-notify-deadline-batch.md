## Обзор

Пуш-пакет из 14 коммитов. Основное — доработка модели уведомлений мессенджера
(заглушение как в Telegram, перенос срока перестал быть «непрочитанным»,
упрощение переключателя до вкл/выкл), навигация панели кнопкой «Назад»,
@-упоминания одиночным выбором, удаление отдельных файлов, а также разовые
фиксы (порядок миграций, копирование списков, устойчивость wazzup-delete).

Детальные расследования и грабли — в `.claude/rules/messenger-ledger.md`
(записи за 2026-07-01). Здесь — сводка того, что уезжает в прод.

## Что вошло

### Модель уведомлений по треду

- **«Заглушить + архив» (как в Telegram).** Заглушённый тред не теряет
  непрочитанное — оно уходит в архивные счётчики `thread_unread_state`
  (`muted_*`) и показывается на новой вкладке **«Заглушённые»** с серым
  бейджем. Прямое @упоминание/ответ тебе «высовываются» в «Непрочитанные»
  (+тост), не снимая mute; прочитал — уехало обратно. Внутри заглушённого
  треда непрочитанное подсвечивается тёмно-серым (тон slate), а не красным.
- **Перенос срока (`change_deadline`) больше НЕ считается непрочитанным** —
  исключён из событийного счётчика (`recompute_thread_unread_for`) и из
  красной подсветки события в ленте. Смена статуса остаётся сигналом.
- **Переключатель уведомлений упрощён до вкл/выкл.** Промежуточный уровень
  «Только сообщения» (`muted_events`) после отключения уведомления о сроке
  стал избыточен → все три точки (колокольчик шапки, меню «⋮», компакт в
  настройках/композере) переведены на простой тумблер. `NotifyLevelControl`
  удалён. В БД техническое состояние `muted_events` осталось inert (из UI не
  выставляется, таких строк 0).
- **Группировка тех-событий ленты.** 3+ подряд идущих прочитанных однотипных
  audit-события сворачиваются в строку-сводку «Срок переносили N раз: X → Y»
  с разворотом. Непрочитанные (красные) в группу не попадают.
- Вкладка **«Заглушённые» третьей** в порядке инбокса: Непрочитанные · Все ·
  Заглушённые · Нужно ответить · Ждём клиента.

### Панель и композер

- **Навигация кнопкой «Назад» браузера.** Открытие треда и переключение
  вкладок боковой панели пишутся в историю браузера (`push` вместо `replace`).
  «Назад»/«Вперёд» (и свайп на мобиле) отматывают треды и вкладки; «Назад» за
  первый тред закрывает панель; на мобильном инбоксе системная «Назад»
  закрывает чат к списку.
- **@-упоминания — одиночный выбор.** Попап `@` вместо мультивыбора с кнопкой
  теперь работает как в Telegram/Slack: клик/Enter сразу вставляет тег и
  закрывает попап, навигация ↑↓. Вставка текста с «@» из буфера больше не
  открывает список (мини-плагин `handlePaste` гасит suggestion на один тик).
- **Копирование нумерованного списка** из бабла больше не превращается в
  маркированный: `wrapOrphanListItems` восстанавливает обёртку `<ol>` (со
  `start`) в `MessageList.handleCopy`.

### Удаление файлов (⏳ бэкенд ждёт ручного деплоя)

- Кнопка «Удалить» у отдельного вложения (у своих) + пункт «Скачать все файлы»
  при 2+ вложениях в меню сообщения.
- Удаление файла/сообщения **в подключённом канале**: новые edge-функции
  `telegram-mtproto-delete`, `wazzup-delete`, `telegram-business-delete`;
  `telegram-delete-message` расширена на массив id; per-file адрес каждого
  файла пишется при отправке (миграция `message_attachments.telegram_message_id`
  / `wazzup_message_id`). Правила подписи: у TG/MTProto/Business при тексте
  файл в канале не трогаем; Wazzup — всегда можно.

### Разовые фиксы

- **Порядок миграций 01.07 закрыт.** 4 миграции `20260701_*` переименованы в
  таймстемпы, совпадающие с прод-историей (`mute 200007 → notify 204432 →
  attachments 205000 → deadline 210436`), чтобы `db push` применял их в верном
  порядке и не перезатирал исключение `change_deadline`; на проде — no-op
  (версии уже в истории).
- **wazzup-delete** — `fetch` к API обёрнут в try/catch → честный `{ok:false}`
  вместо 500 при сбое сети.
- **Единый предикат клиентского треда** (`isClientFacingThread`) — одно
  определение для композера и раскраски баблов (раньше расходились →
  командные сообщения в MTProto не серые).
- **Telegram Business** — сохраняем `@username` собеседника в карточку
  (раньше только `tg:<id>`). Edge уже задеплоен.
- **«Непрочитанные»** — ручной unread сортируется по моменту нажатия кнопки
  (`last_read_at`), а не «всегда сверху».
- **Тумблеры показа иконок и префиксов проектов** в настройках сайдбара
  (пресеты интерфейса; параллельная сессия).

## Статус деплоя

- **Фронт** — уезжает этим пушем через CI/CD (GitHub Actions → Docker →
  VPS blue/green). Завершает фичи, чья БД **уже в проде** (mute-архив, три→два
  уровня, перенос срока, группировка, back-навигация, @-упоминания, сортировка,
  клиент-предикат, тумблеры сайдбара).
- **БД (в проде уже применено через MCP):** `thread_unread_state.muted_*`,
  `recompute_thread_unread_for`, `get_inbox_muted_threads`,
  `get/set_my_thread_notify_level`, `change_deadline`-исключение, сортировка
  непрочитанных. Переименованные файлы миграций на проде — no-op.
- **⏳ Ждёт РУЧНОГО деплоя (фича удаления файлов):** миграция
  `20260701205000_message_attachments_external_ids.sql`; edge
  `telegram-mtproto-delete` / `wazzup-delete` / `telegram-business-delete`
  (verify_jwt default) + `telegram-delete-message` (redeploy `--no-verify-jwt`)
  + `wazzup-send` / `telegram-send-message` (`--no-verify-jwt`); mtproto-service
  (rsync + rebuild). До этого деплоя кнопка «удалить в канале» во фронте вызовет
  ещё не существующую функцию — само сообщение/файл удалится в сервисе, но не в
  канале. Деплой строго вместе + смок по каналам.
