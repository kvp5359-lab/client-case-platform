# Тосты новых сообщений: «Скрыть все» + гашение при прочтении

**Дата:** 2026-07-06
**Тип:** fix (фронт)
**Статус:** ждёт деплоя фронта

---

Три жалобы на всплывающие уведомления (тосты) о новых сообщениях. Все три — чистый
фронт, каналы/БД/edge не тронуты.

## 1. Кнопка «Скрыть все» — нестабильная позиция

**Симптом:** то не отображается, то показывается далеко от верхнего уведомления.

**Корень:** [`DismissAllToasts.tsx`](../../src/components/DismissAllToasts.tsx) вычисляла
позицию, замеряя `getBoundingClientRect()` каждого тоста в момент, который часто ловил
анимацию входа/недомонтаж → кнопка «промигивала» или замеряла промежуточное положение.
В комментарии кода упоминался `ResizeObserver`, но по факту его не было — стоял костыль
`setInterval` каждые 100мс на 600мс.

**Фикс:** позиция пересчитывается через `ResizeObserver` + `MutationObserver` на
контейнере тостера (`[data-sonner-toaster]`) — ловят и добавление/удаление тостов, и
изменение высоты стека (загрузка картинок во вложениях). `setInterval`-костыль убран.
При пустом DOM `pos` не обнуляется (нет мигания).

## 2. «Скрыть все» не кликается под модалкой

**Симптом:** при открытом модальном окне крестик отдельного тоста нажимается, а «Скрыть
все» — нет, хотя она видна.

**Корень:** Radix-модалка (Dialog) при открытии ставит `body { pointer-events: none }` и
пускает клики только внутрь себя. Тосты sonner спасаются CSS-оверрайдом
`[data-sonner-toaster] { pointer-events: auto !important }` — поэтому крестик тоста (он
внутри тостера) кликается. А «Скрыть все» — самостоятельная кнопка **вне** тостера, без
этого оверрайда → видна, но клики не доходят. Плюс `z-[999]` против `999999` у тостера.

**Фикс:** кнопке заданы `pointer-events: auto` (возвращает клики себе даже при
`body{pointer-events:none}`) и `z-index: 1000000` (уровень тостера). Порог показа —
3+ тоста (`MIN_TOASTS`).

## 3. Тосты не гаснут при завершении/удалении треда

**Симптом:** завершил задачу в боковой панели (тред автоматически помечается прочитанным)
— висящие тосты по нему остаются. При удалении — то же.

**Корень:** гашение (`dismissThreadToasts`) висело только на одном хуке
`useMarkThreadReadIfFinal` (завершение через финальный статус); удаление треда
(`useDeleteThread`) тосты не трогало вовсе.

**Фикс (унификация):** гашение повешено на единую точку `markAsRead`
([`messengerReadStatusService.ts`](../../src/services/api/messenger/messengerReadStatusService.ts)),
через которую проходят **все** пути пометки прочитанным (открытие треда, кнопка
«Прочитано», реакция, инбокс, завершение задачи). Удаление треда гасит явно (delete не
помечает прочитанным). Разрозненный вызов из `useMarkThreadReadIfFinal` убран.

**Слои:** реестр тостов (`groupedLines`/`makeGroupKey`/`dismiss{Thread,Project}Toasts`)
вынесен в leaf-модуль [`src/lib/messenger/toastRegistry.ts`](../../src/lib/messenger/toastRegistry.ts),
чтобы сервис `markAsRead` мог его звать без импорта из `hooks/` (направление
зависимостей). `useMessageToastPayload.ts` ре-экспортит примитивы для старых импортёров.

## Грабли (на будущее)

1. Любой плавающий UI **вне** тостера sonner под Radix-модалкой требует
   `pointer-events: auto` на себе — иначе `body{pointer-events:none}` глушит клики даже
   при высоком z-index.
2. «Прочитано → тосты гаснут» теперь идёт через `markAsRead`; новый путь прочтения
   покрывается автоматически — не дублировать `dismissThreadToasts` по хукам.
3. Удаление треда гасит тосты отдельно (не идёт через `markAsRead`).
