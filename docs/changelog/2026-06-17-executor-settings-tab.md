# Вкладка «Настройки» проекта для роли «Исполнитель»

**Дата:** 2026-06-17
**Тип:** feature
**Статус:** completed (БД применена в прод)

---

## Что было

Вкладка «Настройки» проекта (карточка проекта: контакт, поля, интеграции,
участники) не показывалась пользователям с проектной ролью **«Исполнитель»**.
Видимость вкладки определяется `project_roles.module_access.settings`, а у
исполнителя этот флаг был выключен (`false`).

## Что стало

У роли «Исполнитель» включён доступ к вкладке «Настройки».

- **Дефолт для новых воркспейсов:** функция
  `get_project_executor_module_access()` — `settings: false → true`.
- **Бэкфилл существующих ролей:** `UPDATE project_roles … WHERE name='Исполнитель'`
  (на момент применения — 2 роли).

Это даёт исполнителю **видеть** вкладку. Редактирование внутри
(карточка проекта, участники и т.д.) по-прежнему гейтится отдельными
правами `permissions.settings` — они не менялись.

## Применение

Миграция применена в прод через MCP (`apply_migration`). Файл в репо —
`supabase/migrations/20260617_executor_settings_module_access.sql`.
