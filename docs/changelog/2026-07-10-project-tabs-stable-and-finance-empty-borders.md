# Вкладки проекта не скачут по ширине + рамки у пустых секций финансов

**Дата:** 2026-07-10
**Тип:** fix + style (фронт)
**Статус:** деплой (push в main → CI/CD blue/green)

---

Две UI-полировки на странице проекта.

## 1. Ряд вкладок проекта перестал «скакать» по ширине

**Симптом:** ряд вкладок модулей (`ProjectModuleTabBar`) при переключении между
вкладками то занимал всю ширину, то сильно ужимался, оставляя пустое место справа.

**Две причины:**

1. **«Частичная» вкладка растягивалась не всегда.** Последняя видимая вкладка
   растягивалась на остаток места (`partialWidth`, truncate) — но если этой
   последней оказывалась **активная** вкладка, код показывал её по контенту, не
   растягивая (оставляя дыру справа). Когда в хвост попадала неактивная — остаток
   заполнялся, когда активная — нет. Отсюда скачок.
2. **Замер игнорировал «⋮»-меню активной вкладки.** Ширины меряются в скрытом
   ряду-клоне, где рендерятся только иконка + подпись, **без** `renderTabExtra`
   (кнопка «⋮» у Задач/Анкет). У активной вкладки в реальном ряду «⋮» добавляет
   ~34px, которых замер не видел → число влезающих вкладок зависело от того, какая
   вкладка активна (у «Финансов» «⋮» нет, у «Анкет» есть).

**Фикс (выбор поведения — «по контенту, лишнее в бутерброд»):**
- Убрана «частичная» вкладка целиком (`partialWidth`, `showPartial`, спец-кейс
  активной). Все видимые вкладки идут строго по контенту (`shrink-0`), что не
  влезло — уходит в меню «≡». Пустое место справа стабильно, ширина вкладок
  фиксирована и не прыгает.
- Замер теперь учитывает «⋮» активной вкладки: в подсчёт добавляется
  `ACTIVE_EXTRA_PX` для строки клона с `data-tab-id === activeTab`, эффект
  перезамера зависит от `activeTab`. Число видимых вкладок больше не зависит от
  того, какая вкладка активна.

Файл: `src/page-components/ProjectPage/components/ProjectModuleTabBar.tsx`.

## 2. Пустые секции финансов — в пунктирной рамке

**Симптом:** секции «Услуги проекта», «Доходы», «Расходы» при отсутствии записей
показывали «висящий» текст по центру («Доходов пока нет» и т.п.) — выглядело
неаккуратно.

**Фикс:** общий `EmptyState` получил опциональный проп `bordered` — обводит
пустой блок пунктирной рамкой (`border-dashed border-border` + лёгкий фон
`bg-muted/20`). Включён только в финансовых секциях (услуги + транзакции доходов/
расходов). Остальные использования `EmptyState` не затронуты (по умолчанию без
рамки).

Файлы: `src/components/ui/empty-state.tsx`,
`src/components/projects/finance/ProjectServicesSection.tsx`,
`src/components/projects/finance/ProjectTransactionsSection.tsx`.
