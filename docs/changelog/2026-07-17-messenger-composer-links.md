# Полноценная работа со ссылками в композере мессенджера

**Дата:** 2026-07-17
**Тип:** feature
**Статус:** completed (деплой: push в main → CI/CD blue/green; чистый фронт, БД/edge не тронуты)

---

Одна многофункциональная кнопка в тулбаре поля ввода + всплывашка для полного
цикла работы со ссылками. Пайплайн отправки не трогался — `<a>` уже проходил и в
бабл (санитайзер пропускает `href/target/rel`), и в Telegram (конвертер
`_shared/htmlFormatting.ts` сохраняет `<a>`, кейс `link` в parity-тесте).

## Что умеет

- **Прикрепить ссылку к выделенному тексту** — выделил → кнопка 🔗 → ввёл URL.
- **Каретка внутри ссылки** → всплывашка сама показывает URL + «перейти /
  изменить / снять».
- **Выделено несколько ссылок** → поле URL прячется, остаётся одно действие
  «Снять все ссылки» (текст остаётся).
- **«Вставить»** из буфера обмена в пустом поле URL.
- URL без схемы дополняется `https://`.

## Реализация

- [`messengerLinkPopup.tsx`](../../src/components/messenger/messengerLinkPopup.tsx)
  (нов) — единый попап двух режимов (`view` — открывается сам по
  `selectionUpdate`; `edit` — по кнопке тулбара). Портал в body + закрытие по
  mousedown снаружи (паттерн «Номер пункта»). Связь «кнопка → попап» через
  реестр на `globalThis` (`Symbol.for`) — устойчиво к двойной загрузке модуля
  при HMR.
- [`lib/messenger/linkUrl.ts`](../../src/lib/messenger/linkUrl.ts) (нов) — чистые
  `normalizeHref` (whitelist схем `http/https/mailto/tel`; прочее, включая
  `javascript:`, обезврежено префиксом `https://`) + `countLinkSegments`
  (считает отдельные сегменты ссылок, а не уникальные URL). +9 тестов.
- [`MinimalTiptapEditor.tsx`](../../src/components/messenger/MinimalTiptapEditor.tsx) —
  8-я кнопка тулбара + монтаж попапа.
- [`globals.css`](../../src/app/globals.css) — пороги container-query под 8
  кнопок (новый 376px прячет 8-ю и показывает «Aa»).

## Грабли (подробно — `messenger-ledger.md` за 2026-07-16)

- **Кнопка «ничего не делала»:** попап открывался и тут же закрывался — кнопка
  тулбара открывает по `mousedown`, а закрытие «по клику снаружи» слушало тот же
  `mousedown`. Фикс: закрывающий слушатель вешать через `requestAnimationFrame`.
- **HMR-дубль модуля:** реестр «editor → открывашка» перенесён на `globalThis`
  через `Symbol.for` (модульный WeakMap у кнопки и попапа расходился).

## Осознанно не делал

Ретрофит существующего `link-preview-popup.tsx` (у другого, полного
tiptap-редактора; view-режим концептуально дублируется) под оба редактора — он
позиционируется `absolute` относительно редактора, в композере это ломается
трансформ-предком; плюс у него нет создания по выделению/мультиснятия/вставки.
Общая **чистая** логика вынесена в `linkUrl.ts`, презентация раздельна намеренно
(иначе — преждевременное обобщение + риск регрессии чужого редактора).
