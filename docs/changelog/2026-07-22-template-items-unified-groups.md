# Единый список состава шаблона: группы задач и в окне добавления

## Проблема
При создании проекта список «Задачи и чаты» показывал задачи, разложенные по
группам шаблона (как в редакторе шаблона). Кнопка «Добавить из шаблона» в уже
созданном проекте показывала те же задачи ПЛОСКИМ списком — без групп и без
кнопки «Выбрать все». Два окна об одном и том же выглядели по-разному.

## Причина
UI-компонент списка уже был общий (`TemplateItemsList`), но группы задач
грузились отдельным хуком прямо в `CreateProjectDialog` и в диалог добавления
не передавались вовсе.

## Решение
Группы задач шаблона переехали в общий хук состава шаблона
`useProjectTemplateContent` (его и так зовут оба окна). Теперь оба диалога
получают одни и те же данные из одного места и рендерят один и тот же список:

- `AddFromTemplateDialog` получил `taskGroups` → задачи группируются;
- туда же добавлена кнопка «Выбрать все / Снять все» — была только при создании;
- `CreateProjectDialog` перестал грузить группы сам (дубль убран).

Группа, все задачи которой уже добавлены в проект, не показывается — пустые
группы список скрывает сам.

- `src/components/projects/AddFromTemplateDialog.tsx`
- `src/components/projects/CreateProjectDialog.tsx`

## Продолжение: группы не появлялись в списке задач после добавления

**Замер (не догадка):** в базе после добавления всё верно — группа
`project_task_groups` создана, у задач проставлен `task_group_id`. То есть
движок посева (`seedProjectContent`) отработал одинаково для обоих путей.

**Причина была в кэше экрана.** Список задач берёт группы из отдельных
запросов (`task-groups`, `task-group-membership`), а окно «Добавить из шаблона»
сбрасывало только план/наборы/анкеты/треды. Группы приезжали лишь после
перезагрузки страницы.

**Решение:** список ключей, которые надо сбросить после посева, вынесен в один
модуль `services/projects/invalidateAfterSeed.ts` рядом с самим движком. Новый
срез данных у посева — добавляется туда, а не по окнам.

**Заодно:** в режиме добавления группы шаблона теперь переиспользуются по
имени, а не создаются заново — иначе повторное «Добавить из шаблона» плодило
бы копии одной и той же группы.
