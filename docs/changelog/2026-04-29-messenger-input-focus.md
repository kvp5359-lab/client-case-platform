# Возврат фокуса в поле ввода после отправки сообщения

**Дата:** 2026-04-29
**Тип:** fix
**Статус:** completed

---

## Контекст

Пользователь жалуется: в чате после нажатия «Отправить» курсор пропадает из поля ввода. Чтобы продолжить набирать следующее сообщение, приходится кликать в поле мышкой. Особенно мешает, когда нужно подряд скинуть несколько коротких реплик.

## Причина

На время отправки `useSendMessage` поднимает флаг `isPending=true`. Этот флаг прокидывается в `MinimalTiptapEditor` как `disabled`, а внутри редактора синхронизируется в Tiptap через `editor.setEditable(false)`. Когда редактор становится неактивным, браузер снимает с него фокус. После успеха `isPending` возвращается в `false`, редактор снова доступен — но фокус никто не возвращает.

Простое `editor.commands.focus('end')` сразу после `clearContent()` в `handleSend` не помогает: `isPending` ещё `true`, редактор disabled, фокус не закрепляется.

## Решение

В `MessageInput.tsx` добавлен эффект, который ловит переход `isPending: true → false` и в этот момент возвращает фокус в редактор. Используется `useRef` для хранения предыдущего значения, чтобы не перехватывать первый рендер с `isPending=false`.

```tsx
const wasPendingRef = useRef(false)
useEffect(() => {
  if (wasPendingRef.current && !isPending) {
    editorRef.current?.commands.focus('end')
  }
  wasPendingRef.current = isPending
}, [isPending])
```

## Файлы

- `src/components/messenger/MessageInput.tsx` — эффект на падающий фронт `isPending`

## Почему так

**Возврат фокуса по факту разблокировки редактора, а не по событию отправки.** Если повесить фокус на колбэк `onSend`, фокус упадёт в момент, когда редактор ещё disabled, и Tiptap его не зафиксирует. Привязка к `isPending → false` гарантирует, что в момент вызова `focus()` редактор уже принимает фокус.
