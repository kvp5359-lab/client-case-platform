## 1. Белый блок в настройках при открытой боковой панели (fix, регрессия)

**Симптом.** На странице настроек (и других с вложенным `<main>`) при открытой
панели контент схлопывался в узкую полоску, остальное — большой белый блок.

**Причина (замер геометрии).** Правило push-отступа панели (введено в этот же
день) таргетило селектором `body[data-panel-open] main` — он матчит **любой**
`<main>`. Несколько страниц (настройки, домашняя воркспейса, редакторы шаблонов)
рендерят **свой вложенный `<main>`** внутри app-level. Вложенному тоже вешался
`margin-right: 633px`, и он схлопывался (app-main шириной 561px − nav − 633px →
~48px). Старый селектор был так же широк, но push стоял только на
Задачах/Документах/Финансах (без вложенного main), поэтому не проявлялось.

**Решение.** Маркер `data-app-main` на app-level `<main>` (WorkspaceLayout);
CSS-отступ таргетит `main[data-app-main]` — только оболочку, не вложенные mains.
Замер после фикса: вложенный main `margin-right: 0`, заполняет суженный app-main.
Фикс общий — лечит все страницы с вложенным `<main>`.

## 2. Пропадали задачи в настройках шаблона проекта (fix, архитектурный)

**Симптом.** В шаблоне «СР-Испания» часть задач исчезла из редактора (badge 21,
видно меньше). «Пропали, когда добавил первую группу задач».

**Причина (замер по данным прода).** `thread_templates.task_group_id` хранился
**глобально**, а задача-«рыба» **шарится** между проект-шаблонами через junction
`project_template_thread_templates`. Группа же (`project_template_task_groups`)
принадлежит ОДНОМУ проект-шаблону. Когда 5 общих задач сгруппировали под
«Написание БП» в шаблоне «Бизнес-план», их глобальный `task_group_id` прописался
всем; в «СР-Испания» эта группа чужая, а редактор рисует только группы своего
шаблона → задачи не отображались. **Данные не терялись** — были скрыты.

**Решение (перенос на junction).** Группировка задач-шаблонов переехала на
`project_template_thread_templates.task_group_id` (пер-проект-шаблон): каждый
шаблон группирует общую задачу независимо.
- Миграция: колонка на junction (FK → `project_template_task_groups`,
  `ON DELETE SET NULL`) + индекс `(template_id, task_group_id)` + бэкфилл
  (переносит группу ТОЛЬКО там, где она принадлежит этому шаблону) → скрытые
  задачи восстановлены (в «СР» стали несгруппированными, в «Бизнес-план» остались
  в «Написание БП»). Бэкфилл идемпотентен и есть само восстановление.
- `mapJunctionRow` отдаёт пер-проектную группу (перебивает глобальное поле
  «рыбы») → редактор и create-project читают её; `assignThread` пишет в junction
  (scoped по `template_id + thread_template_id`).
- `thread_templates.task_group_id` **оставлен deprecated** на время выката фронта
  (задеплоенный прод-фронт ещё читает глобальное поле).

**Проверено вживую:** все 19 задач видны в редакторе «СР-Испания» (5
восстановленных без группы); запись группы общей задаче в одном шаблоне НЕ трогает
другие (Бизнес-план/консульство сверены по БД, тестовая запись откачена).

**Правки по ревью:** добавлена FK-связь `task_group_id` в `Relationships`
(database.ts); `mapJunctionRow` экспортирована и покрыта тестом (+5) — защищает
инвариант «пер-проектная группа перебивает глобальную».
