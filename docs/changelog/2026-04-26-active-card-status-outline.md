# Подсветка активной карточки на досках — обводка цветом статуса

**Дата:** 2026-04-26
**Тип:** UX polish
**Статус:** completed

---

## Контекст

На досках карточка треда/проекта, открытая сейчас в боковой панели, подсвечивалась жёлтым фоном (`bg-brand-100`) и голубой рамкой (`border-brand-200`), имя становилось жирным и красилось в `text-brand-700`, иконка папки — в `text-brand-600`. На фоне статусных цветов колонок выглядело шумно и «перебивало» собственный цвет статуса карточки.

## Изменение

Активная карточка теперь **никак не меняется внутри** — те же цвета, тот же шрифт. Вокруг неё рисуется **обводка толщиной 3px цвета её статуса**:

- для треда — `task.status_color` (через `safeCssColor`)
- для проекта — цвет статуса из справочника воркспейса
- fallback, если статус без цвета — бренд-цвет (`hsl(var(--brand-500))`)

Реализовано через inline `outline: 3px solid <color>` + `outlineOffset: -3px`, поэтому обводка ложится **внутрь** границ карточки и не сдвигает раскладку соседних элементов.

Применено в обоих режимах отображения списков на досках — `cards` и `list`.

## Файлы

- `src/components/boards/BoardTaskRow.tsx` — обводка треда + удалены `font-medium text-brand-700` с имени
- `src/components/boards/BoardProjectRow.tsx` — обводка проекта + удалены `text-brand-600/700` с иконки/имени/next_task

## Почему так

Цвет статуса — самая информативная характеристика карточки. Использовать его же для маркера «открыто сейчас» даёт двойной сигнал без визуального шума: пользователь видит и статус, и активное состояние одним и тем же цветом. Внутренние цвета не трогаем, чтобы статусные колонки и раскраска полей не «прыгали» при смене активной карточки.
