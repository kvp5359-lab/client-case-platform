## 1. Пакетное управление исполнителями в списках (feat)

Тулбар пакетных действий для `item_lists` переделан: вместо двух отдельных
кнопок («Сменить статус» + «В корзину») — единое меню **«Действия»**. Для
списков проектов добавлено управление исполнителями сразу по всей выделенной
группе.

**Что нового:**

- Меню «Действия» с подменю «Сменить статус» (как было, но свёрнуто в submenu),
  пунктом «В корзину» и — только для проектов — блоком исполнителей:
  - **Добавить исполнителей** — выбор участников через общий
    `ParticipantsPicker`, роль «Исполнитель» добавляется в каждый выделенный
    проект.
  - **Отстранить исполнителя** — список-объединение исполнителей всех выделенных
    проектов (с подсчётом «в N проектах»), снятие одного выбранного.
  - **Отстранить всех исполнителей** — снимает роль «Исполнитель» во всех
    выделенных проектах (с confirm).

**Логика исполнителей** (`bulkExecutorActions.ts`): исполнитель = участник
`project_participants` с ролью `Исполнитель` в массиве `project_roles`. Повторяет
семантику `applyRoleParticipantsChange` из `useProjectParticipantsMutations`, но
работает сразу по нескольким проектам. Добавление: если строки участника в
проекте нет — INSERT, если есть без роли — UPDATE с дозаписью роли. Снятие: если
после снятия роли других ролей не осталось — строка удаляется целиком, иначе
UPDATE.

## 2. Дублирующиеся иконки списка задач в правой панели (fix)

**Было:** при перекликивании между тредами разных проектов на доске
«Входящие» / `/boards` в правой боковой панели периодически появлялось
несколько одинаковых иконок «Список задач». Они не кликались (вели в чужой
проект) и пропадали после перезагрузки страницы.

**Причина:** вкладка задач привязана к проекту (id = `tasks:<projectId>`). При
быстром переключении между тредами разных проектов панель переключает scope, и
из-за гонки (scope-switch) tasks-вкладка прошлого проекта успевает попасть в
набор вкладок нового. Запись в `task_panel_tabs` от этого защищена раньше
(коммит `a53a6f0`, race-guard + `sanitizeTabsForScope`) — поэтому мусор не
сохранялся в БД и исчезал после reload, но на экране в текущей сессии висел.

**Стало:** страховочная очистка вкладок в `TaskPanelTabbedShell` (раньше убирала
только тред-вкладки чужого scope) теперь так же убирает tasks-вкладки, чей
`refId` указывает на другой проект. Лишняя иконка исчезает сразу, не дожидаясь
перезагрузки. Легитимную вкладку текущего проекта и legacy-записи без `refId`
(фолбэк на `projectId` в рендере) не трогает.

**Затронутые файлы:**
- [`TaskPanelTabbedShell.tsx`](../../src/components/tasks/TaskPanelTabbedShell.tsx)
  — расширен cleanup-эффект.
