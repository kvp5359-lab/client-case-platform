# «Скачать все файлы» из сообщения — одним ZIP-архивом

**Дата:** 2026-07-02
**Тип:** feat (UI)
**Статус:** completed (ждёт деплоя фронта)

---

## Проблема

Пункт «Скачать все файлы» в меню сообщения качал вложения **по одному**
(несколько загрузок подряд с паузой 400мс, чтобы браузер не резал пакет).
Неудобно при 5–10 файлах.

## Что сделано

Теперь собираем все вложения в **один ZIP** и отдаём одной загрузкой
`вложения.zip`.

- В сервис добавлен `fetchAttachmentBlob(storagePath, fileId)` — отдаёт сырой
  `Blob`; `downloadAttachmentBlob` стал обёрткой над ним (поведение прежнее).
- `handleDownloadAll` (`MessageMenuBody`) качает блобы, кладёт в `JSZip`
  (уже был в зависимостях — новый пакет не нужен) и отдаёт архив.
- `jszip` грузится **динамически** (`await import('jszip')`) — не тянется в
  бандл меню.
- **Уникализация имён**: одинаковые имена в сообщении → `имя (2).pdf` и т.д.
  (иначе zip перезаписал бы файл).
- Ошибки: ни один не скачался → «Не удалось скачать файлы»; часть не добавилась
  → «Не удалось добавить в архив файлов: N».

Пункт по-прежнему виден только при 2+ вложениях. Пауза между загрузками больше
не нужна (загрузка одна).
