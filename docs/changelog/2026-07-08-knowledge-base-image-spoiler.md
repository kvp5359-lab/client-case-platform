# База знаний: инлайновая «картинка-подсказка» (image spoiler) + переиспользование настроек картинки

**Дата:** 2026-07-08
**Тип:** feat + refactor (фронт)
**Статус:** деплой (push в main → CI/CD blue/green)

---

Новый способ вставки картинки в статью базы знаний: **инлайновый значок-«подсказка»**
внутри текста, по клику разворачивается картинка блоком под строкой и сворачивается
обратно. В отличие от существующего блочного спойлера (`accordion`, большой блок на
всю строку) — это лёгкая инлайновая «помогалка» рядом со словом.

## 1. Что это и как устроено

- Значок — чип с **редактируемой подписью** (по умолчанию «подсказка») прямо в потоке
  текста; ставится и в конце строки, и в середине абзаца.
- Клик по чипу → картинка раскрывается блоком под строкой, повторный клик → сворачивает.
- Не перекрывает текст (не оверлей), не закрывается по клику мимо, по умолчанию свёрнуто.

**Ключевое решение — checkbox-hack, а не нативный `<details>`.** Статья вне редактора
рендерится как сырой HTML (`dangerouslySetInnerHTML`, без React/Tiptap). `<details>` —
flow-content, внутри `<p>` браузер-парсер **разорвал бы абзац**. Поэтому нода
сериализуется в phrasing-элементы (`span` / `input[type=checkbox]` / `label` / `img`),
которые валидны внутри `<p>`, а сворачивание в статике держит чистый CSS
`:checked ~ .image-spoiler-media` — без JS. Проверено: через `sanitizeHtml`
(дефолтный DOMPurify) структура проходит целиком и `<p>` не рвётся.

**Ловушка верстки (была и починена):** сначала `.image-spoiler` был `inline-block` —
при раскрытии его ширина становилась равной ширине картинки, чип не влезал после
текста и переносился на новую строку целиком. Стало `display: inline` (чип остаётся
узким на месте, картинка падает блоком ниже).

## 2. Переиспользование настроек картинки (рефактор ImageBlock)

Настройки картинки-подсказки (Alt / подпись / высота / ширина / скругление / контур /
тень) — **те же, что у обычной `ImageBlock`**, а не отдельная реализация. Для этого
общая часть вынесена в новый [`image-shared.tsx`](../../src/components/tiptap-editor/node-views/image-shared.tsx):

- карты стилей (`roundedClasses` / `shadowStyles` / `borderWidthValues` / `sizeConfig`) —
  единый источник вместо трёх копий (были в `image-block.ts`, `image-block-view.tsx`);
- `imageFigureSpec(attrs)` — единый рендер статичного `<figure data-type="image-block">`
  для ImageBlock и раскрытой подсказки;
- `ImageSettingsFields` — общая форма настроек (рендерится в Popover обоих node-view).

`image-block.ts` (renderHTML) и `image-block-view.tsx` (тело попапа) переведены на общий
модуль → −249 строк дублей. **Регрессий у обычных картинок нет:** замером сверено, что
`imageFigureSpec` даёт байт-в-байт тот же HTML, что старый код (3 набора атрибутов, JSON-diff).

## 3. Fix: «двойной клик» при смене размера/тени

**Симптом:** смена ширины/высоты/тени сворачивала картинку; применялось только со
второго клика.

**Корень:** node-view подсказки пере-инициализируется при смене атрибута (ре-парс /
ремоунт), локальный `useState(open)` сбрасывался в `false` → картинка сворачивалась.

**Фикс** ([`image-spoiler-view.tsx`](../../src/components/tiptap-editor/node-views/image-spoiler-view.tsx)):
`open` и состояние попапа настроек вынесены из React-стейта в модульный стор по `uid`
ноды. Переживает и ремоунт node-view, и полный ре-парс контента (`uid` стабилен и
round-trip'ится через `data-uid`). Документ при этом не пачкается — «раскрыто» нигде
в HTML не сохраняется, статья всегда грузится свёрнутой.

## Грабли (на будущее)

- Инлайновый узел, который должен работать в **статичном** HTML (вне редактора), нельзя
  строить на `<details>` — он flow-content и рвёт `<p>`. Использовать phrasing-элементы
  + CSS `:checked` (checkbox-hack).
- `inline-block` у инлайнового значка с широким раскрываемым контентом → значок
  «толстеет» до ширины контента и переносится. Для «значок на месте, контент блоком
  ниже» — `display: inline` + блочный media.
- Локальный `useState` в Tiptap node-view **не переживает** смену атрибутов (ремоунт/
  ре-парс). Состояние, которое должно жить между правками (раскрыто/попап) — держать в
  модульном сторе по стабильному id ноды, не в документе.
- Настройки/стили картинки — единый источник в `image-shared.tsx`. Новый потребитель
  (ещё один вид картинки) — оттуда, не копировать карты стилей.
