# Вкладки модулей проекта: адаптация под видимую ширину при открытой панели

**Дата:** 2026-07-05
**Тип:** fix (UI, фронт-CSS)
**Статус:** completed (ждёт деплоя фронта)

---

## Проблема

Ряд вкладок модулей проекта (`⚙️ / Задачи / Документы / Анкеты / Подбор ВНЖ /
Материалы / …` в `ProjectPage`) не помещался в видимую область, когда открыта
боковая панель треда. Подписи вкладок скрывались только по **viewport**-брейкпоинту
(`hidden md:inline`), а панель сужает область контента, не меняя ширину окна →
на десктопе (viewport ≥ md) все подписи оставались, и правый край ряда уезжал под
панель / обрезался.

## Решение

Адаптация по **реальной ширине ряда** (CSS container query — как уже сделано в
композере и на вкладке «Финансы»), а не по ширине окна.

- Ряд вкладок стал контейнером (`.project-tabs-cq` → `container: proj-tabs /
  inline-size`).
- `TabsList` → `max-w-full`; вкладки — `min-w-0` (могут ужиматься ниже своего
  контента), иконки и меню-«⋮» — `shrink-0`.
- **Активная вкладка — `shrink-0` + `whitespace-nowrap`**: подпись всегда видна
  целиком.
- **Неактивные вкладки** отдают ширину: подпись `truncate min-w-0` (обрезается
  многоточием при нехватке места), а при сильной тесноте (container ≤ 560px)
  скрывается совсем — остаётся иконка (`.proj-tab-label--secondary` →
  `display:none`).

## Ловушка: push- vs overlay-режим панели

Боковая панель работает в двух режимах, и это влияло на ширину контейнера:

- **push** (вкладки со списком задач — `TaskListView`/`Финансы`/`Документы`):
  `body[data-panel-open][data-panel-mode="push"] main { margin-right:
  var(--panel-width) }` реально сужает `<main>` → container query видит
  правильную ширину сам.
- **overlay** (Материалы/Анкеты/Подбор ВНЖ/История/Дневник): панель **накрывает**
  контент, `<main>` остаётся во всю ширину → ряд «думал», что у него полная
  ширина, ничего не сворачивал, правый край уходил под панель.

Фикс для overlay: при открытой панели и **не** push-режиме резервируем ширину
панели для самого ряда вкладок —
`body[data-panel-open]:not([data-panel-mode='push']) .project-tabs-cq {
margin-right: var(--panel-width) }` (только `@media (min-width: 768px)`; на мобиле
панель fullscreen, ряд не виден). В push-режиме правило не применяется — там
`<main>` уже сужен, иначе было бы двойное вычитание.
