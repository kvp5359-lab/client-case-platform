## 1. Серая подсказка внутри текстового поля (из описания)

**Было:** «Описание поля» было видно только в тултипе по иконке `(?)` рядом с
заголовком. В самом поле подсказки не было.

**Стало:** при фокусе в текстовом поле внутри показывается серым текстом
подсказка — свой `placeholder` поля, а если он не задан, то текст из «Описание
поля». Описание при этом остаётся и в тултипе `(?)`.

- Применяется только к текстовым типам: `text`, `textarea`, `email`, `phone`, `url`.
- Подсказка показывается **только в фокусе**: тогда плавающий заголовок уходит на
  рамку и освобождает тело поля. Без фокуса заголовок стоит внутри (по центру у
  однострочного, вверху у многострочного) и подсказка бы на него наложилась.
- Переносы строк в подсказке схлопываются в один пробел — в HTML `placeholder`
  переносы не отображаются, из-за чего слова слипались («квартира\nГотов» →
  «квартираГотов»).

Файлы:
- [`SimpleInput.tsx`](../../src/components/forms/SimpleInput.tsx): пропсы
  `placeholder` и `placeholderVisible` у текстового `<input>` и `AutoGrowTextarea`;
  цвет плейсхолдера `gray-400` при видимости, иначе `transparent`.
- [`FieldsGrid.tsx`](../../src/components/forms/FieldsGrid.tsx): вычисление
  подсказки (`placeholder` поля → иначе `description`, с заменой переносов на
  пробел) только для текстовых типов; проброс `placeholderVisible={isFocused}`
  через render-prop `FloatingField`.

## 2. Заголовок «уезжал» у поля рядом с многострочным

**Было:** в строке сетки однострочное поле растягивалось по высоте соседнего
многострочного (CSS grid `align-items: stretch`), из-за чего его заголовок
смещался и появлялось пустое пространство.

**Стало:** добавлен `items-start` на обе сетки полей — каждое поле своей
естественной высоты, не тянется под высоту соседа.

- [`FieldsGrid.tsx`](../../src/components/forms/FieldsGrid.tsx): `items-start` на
  основном гриде (6 колонок) и на двухколоночной группе textarea.

## 3. Удаление бэкап-таблицы Telegram (security advisor)

**Было:** таблица `public._backup_project_telegram_chats_20260528` (снимок 28 мая
при унификации Telegram-ботов) осталась в схеме `public` без RLS → Supabase
Security Advisor слал еженедельное предупреждение `rls_disabled_in_public`
(ERROR): таблица потенциально доступна по публичному ключу проекта.

**Стало:** бэкап удалён. Живая таблица `project_telegram_chats` не тронута
(работает, RLS включён).
