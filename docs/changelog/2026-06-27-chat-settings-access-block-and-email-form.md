# Реорганизация настроек чата: блок «Доступ и подписки» + правки email-формы

**Дата:** 2026-06-27
**Тип:** ui / refactor
**Статус:** completed (ждёт деплоя фронта)

---

> Чистый фронт. Логика подписок/доступа/каналов НЕ менялась — только перекомпоновка
> UI и сплит компонентов. tsc 0, lint 0, 753 теста зелёные.

## Что было

В диалоге настроек чата три секции шли подряд и постоянно занимали место:
«Исполнители», «Кто видит чат», «Уведомления» (личный тумблер + шестерёнка
управления подписчиками). Email-форма создания была разрознена: лейбл «Письмо»
с полями получателя/темы в одном месте, отдельная секция «Первое сообщение» —
в другом. Плюс баг: при переключении вкладок Задача/Чат/Email и возврате на
Email цвет значка письма становился красным вместо дефолтного.

## Что стало

### Блок «Доступ и подписки»

- Сворачиваемый блок (свёрнут по умолчанию — нужен реже, чем «Исполнители»).
  В свёрнутом виде — краткая сводка «👁 {кто видит}» (Все участники / роли /
  N участников).
- Развёрнутый — «Кто видит чат» + список подписчиков (для владельца/менеджера).
- «Исполнители» остались отдельной секцией сверху (видны всегда).
- Личная подписка вынесена компактной ghost-кнопкой «Подписаться/Отписаться»
  рядом с «Подключить канал» — видна без разворачивания, минимум места.

`ChatSettingsNotifications` разбит на варианты: `compact` (ghost-кнопка личной
подписки), `personal` (тумблер в рамке), `manage` (поле подписчиков, всегда
развёрнуто внутри блока, только при `canManage`), `full` (legacy). Хук
`useThreadSubscribers` грузится лениво — только когда блок развёрнут.

### Email-форма (создание)

- Получатель + тема + текст объединены в единый блок «Письмо» (рамка + лёгкий
  фон + заголовок с иконкой). Заголовок не «Письмо клиенту» — письмо может быть
  и не клиенту.
- «Кому:» / «Тема:» — в Gmail-стиле: лейбл-префикс внутри поля, с двоеточием,
  размер `text-sm`. Плейсхолдеры убраны.
- «Текст письма» — без отдельного заголовка (только плейсхолдер в поле),
  базовая высота увеличена до ~76px (≈3 строки). Добавлен проп `editorMinHeight`
  (ComposeField → MinimalTiptapEditor).
- Для вкладок «Задача»/«Чат» раскладка и отступы не меняются (рамка только в
  email-создании).

### Фикс цвета значка при переключении вкладок

При переключении Задача/Чат/Email цвет+иконка хардкодились на каждое
переключение, затирая вид, применённый при открытии (дефолт из шаблона/канала).
Возврат на Email делал значок красным (`'rose'`). Фикс: ref-карта `modeLooks`
запоминает иконку+цвет покидаемой вкладки и восстанавливает при возврате
(иначе — дефолт режима); сбрасывается при каждом открытии диалога.

> Глубинно create-диалог всё ещё НЕ читает `workspaces.channel_defaults` (дефолт
> email — хардкод `'rose'`). Отдельная задача; remember-карта лечит симптом
> переключения.
