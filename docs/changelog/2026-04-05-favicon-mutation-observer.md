# Favicon-бейдж не пропадает при навигации

**Дата:** 2026-04-05
**Тип:** fix
**Статус:** completed

---

## Что сделано

- `src/hooks/messenger/useFaviconBadge.ts` — добавлен `MutationObserver` на `<head>`: следит за добавлением новых `<link rel=icon>` и удаляет их, сохраняя только наш динамический тег
- Раньше при клиентской навигации между проектами Next.js снова вставлял свой favicon-тег из `src/app/favicon.ico`, и браузер выбирал его (последний вставленный), а наш бейдж затирался
- Предыдущая реализация чистила чужие теги только при изменении числа непрочитанных — при смене проекта счётчик не менялся, поэтому очистка не срабатывала, и Next-тег побеждал
