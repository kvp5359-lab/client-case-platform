# Капремонт списков, объединение с досками, разделы и роль-колонки проектов

**Дата:** 2026-06-17
**Тип:** feat + perf
**Статус:** completed (БД в проде, фронт ждёт деплоя)

Большой пласт по разделу «Доски и списки»: производительность и UX списков,
слияние раздела «Списки» с «Досками», новая сущность «Разделы», роль-колонки
участников проектов (в списках и карточках досок).

---

## 1. Производительность и загрузка списков

- **Серверная фильтрация** (паритет с досками): списки больше не грузят весь
  воркспейс. `useListThreads`/`useListProjects` (`useListData.ts`) переиспользуют
  инфраструктуру досок — RPC `get_board_filtered_threads/projects`,
  `boardFilterService`, кэш `boardFilteredKeys`. Список = один фильтр →
  `lowerFilterForServer`. Клиентский `useFilteredTasks/Projects` дорезает точно.
- **Настоящая пагинация по скроллу** (Вариант A): `useInfiniteQuery`, страницы по
  60, серверная сортировка через PostgREST `.order()/.range()` поверх board-RPC
  (без новой миграции). Исполнители грузятся только для загруженных строк.
- **Виртуализация строк** (`@tanstack/react-virtual`) в `TableShell` (tbody-spacer,
  `measureElement` по `data-index`); sticky-заголовок, ресайз, colgroup сохранены.
- **Оптимистичный патч** статуса/срока в кэшах `['workspace-threads', …]`
  (`useTaskMutations`) — мгновенный апдейт ячейки; улучшает и доски/задачи.
- Чинит баг-лимит 1000 проектов (`useAccessibleProjects` без пагинации);
  `next_task_deadline` у проектов теперь заполняется.

## 2. UX списков

- Клик по строке открывает **боковую панель** (как на досках — `openThread`/
  `openProject`).
- **Inline-пикеры** срока (`DeadlinePopover` с пресетами) и исполнителей
  (`AssigneesPopover`); колонка исполнителей — режим «Аватарки / Имена».
- **Умные даты** (`formatSmartDateCompact`): сегодня → время, вчера, день недели.
- **Иконка чата** после названия у тредов-НЕ-задач (переиспользует
  `getChatIconComponent` + `COLOR_TEXT`).
- **Быстрый фильтр по заголовку** (`useQuickFilters`, `ColumnFilterPopover`) —
  значения только из текущего списка.
- **Клавиатура** (↑↓/Enter/Space), **переключатель плотности** (localStorage),
  **undo** на inline-правках статуса/срока, **shift-выделение** диапазона
  чекбоксами, **мин. ширина колонок 30px**.
- **Смена типа списка** (Треды/Проекты) в настройках; панель выделения
  закреплена снизу; в редакторе колонок глаз слева, скрытые серым.

## 3. Пакетные действия для тредов

В меню «Действия» списков тредов: добавить исполнителей, отстранить конкретных
/ всех исполнителей, убрать из участников (закрыть доступ — `project_thread_members`),
установить срок. `bulkThreadActions.ts`, `BulkRemovePeopleDialog`, `BulkDeadlineDialog`.

## 4. Объединение «Доски и списки»

Раздел «Списки» упразднён — доски и списки в одном таб-баре `/boards`
(`BoardsPage` рендерит оба типа; `BoardTabContent` / `ItemListTabContent`).
Роутинг: доски `/boards/<uuid>`, списки `/boards/list-<uuid>`; старые `/lists`,
`/lists/[listId]` → редирект. Сайдбар: пункт «Списки» убран, «Доски» → «Доски и
списки». Кнопка «+» → меню «Новая доска» / «Новый список».

## 5. Разделы (новая сущность)

Контейнер уровня воркспейса, группирующий доски и списки (многие-ко-многим).
Например раздел «CRM» = только доски/списки продаж.

- **БД (миграция `20260616_workspace_sections.sql`, уже в проде):**
  `workspace_sections` + `workspace_section_items` (junction) + RLS (видят
  участники, правят владелец/менеджер; anon без доступа). Типы в `database.ts`.
- **Фронт:** хуки `useSections` (CRUD + членство, карты раздел↔элемент);
  раздел — slot-пункт сайдбара (тип `section`, добавляется в настройках сайдбара,
  групповой пункт «Раздел» с кнопкой «создать»); `/boards?section=<id>` сужает
  таб-бар до членов; назначение в разделы из меню «⋯» доски/списка.

## 6. Роль-колонки участников проектов

В списках **и** карточках досок проектов: поля **Исполнители / Администраторы /
Клиенты / Наблюдатели** (аватарки участников с соответствующей `project_roles`).

- Общий хук `useProjectPeopleByRole` (карта `${projectId}:${role}` → участники).
- Списки: колонки + **быстрый фильтр** по ним + **inline-смена роли** из ячейки
  (`AssigneesPopover` controlled + `toggleProjectRole`).
- Карточки досок: те же роль-поля (`listSettingsConfigs.PROJECT_ROLE_FIELDS`,
  `BoardProjectRow`).
- ⚠️ Соответствие: «Наблюдатели» (UI) = роль **`Участник`** (БД) — единое для
  списков и досок.
