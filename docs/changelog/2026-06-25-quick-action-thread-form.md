# Быстрое действие «Задача/Чат»: форма создания треда + иконка из шаблона

**Дата:** 2026-06-25
**Тип:** feat
**Статус:** completed (ждёт деплоя фронта)

---

> Правка из параллельной сессии, вкоммичена вместе с мобильной адаптацией.

## Что было

Быстрое действие создания треда (`QuickAction` kind `new_thread`) создавало тред
**моментально**: брало шаблон, прогоняло `applyTemplate` с пустыми участниками и
сразу `createThread` — без возможности поправить название, срок, исполнителя,
получателей.

## Что стало

### `QuickActionsProvider.tsx`
Быстрое действие «Задача/Чат» теперь **открывает форму создания треда**
(`ChatSettingsDialog`) с предзаполненными данными шаблона и целевым проектом.
Пользователь правит поля (название, срок `start_at`/`end_at`, исполнители,
участники, получатели email, статус) и подтверждает — создание идёт по результату
формы (`ChatSettingsResult`), а не из «сырого» `applyTemplate`.

- `runNewThread` больше не создаёт тред, а загружает шаблон и открывает диалог
  (`threadDialog` state: `{ open, template, targetProjectId }`).
- Новый `handleCreateThread(result)` → `createThread.mutate(...)` с полями из
  формы; на успехе — тост и закрытие диалога, на ошибке — тост с описанием.
- Убран импорт `applyTemplate` (больше не используется здесь).

### `QuickActionsEditor.tsx` (настройка быстрого действия)
- Иконку действия можно выбрать **вручную** — флаг `iconTouched`. После ручного
  выбора автоподстановка иконки больше её не перетирает (раньше смена «вида»
  действия сбрасывала иконку на дефолтную).
- Если иконку не трогали — при выборе **шаблона треда** иконка действия
  автоподставляется из иконки шаблона (запрос `thread_templates` дополнен полем
  `icon`).
