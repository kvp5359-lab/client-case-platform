# Фикс: спойлер (аккордеон) в статье БЗ обрезал высокий контент

**Дата:** 2026-07-01
**Тип:** fix
**Статус:** completed (ждёт деплоя фронта)

---

## Проблема

В редакторе статьи базы знаний раскрытый спойлер (аккордеон) прятал часть
контента: при большом объёме (несколько шагов со скриншотами) низ текста не
помещался и оставался скрыт внутри блока.

## Корень

Нодвью аккордеона (`accordion-view.tsx`) при раскрытии задавал фиксированный
`max-h-[2000px]` + `overflow-hidden` (для анимации сворачивания). Контент выше
2000px обрезался. Read-only просмотр статьи (native `<details>` + CSS в
globals.css) лимита не имеет — там текст виден целиком; проблема только в
редакторе.

## Что сделано

Высоту раскрытого контента больше не капим фиксированным значением. Через ref
измеряем реальную `scrollHeight`, анимируем `0 → высота` при раскрытии, затем
снимаем лимит (`max-height: none`) — контент виден целиком и не режется даже
когда изображения догружаются и высота растёт. Анимация сворачивания/
разворачивания сохранена (`transition-[max-height,opacity]`).
