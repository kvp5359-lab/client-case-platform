# Мобильная адаптация: панель, сайдбар, шапка чата, верхние вкладки

**Дата:** 2026-06-25
**Тип:** fix/feat
**Статус:** completed (ждёт деплоя фронта)

---

## Проблема

На телефоне обнаружились накопившиеся проблемы вёрстки и юзабилити (по скринам
пользователя):

1. **В открытом чате не видно поле ввода.** Боковая панель имела высоту
   `100vh`. На мобильных браузерах `100vh` включает зону адресной строки/нижней
   панели → низ панели (композер) уезжал под хром браузера.
2. **Кнопка «Прочитано» наслаивалась на переключатель режимов видимости.**
   Кнопка была спозиционирована `absolute left-1/2` — центр игнорировал ширину
   соседних элементов, на узком экране они накладывались.
3. **Боковая панель не на весь экран.** Фиксированная десктопная ширина
   (`--panel-width`, 600px) на телефоне оставляла торчащие по краям куски
   списка/соседней панели.
4. **Шапка чата перегружена.** Меню «⋮» и кнопка «+» (добавить доску) были
   спрятаны под `group-hover` → на тач-экране без ховера недоступны вовсе.
   Подписи чипов «Срок»/«Проект» занимали лишнюю ширину.
5. **Мобильный бургер наезжал на верхние вкладки** на `/boards` — активная
   вкладка уходила под кнопку меню.
6. **Горизонтальные списки листались без признаков прокрутки** — полоса вкладок
   досок и сегменты «Входящих» («Ждём…» обрезался у края).
7. **Мелкие тач-цели** у icon-кнопок сайдбара (30px).

## Что сделано

Всё — чистый фронт (CSS и классы), через `md:`-брейкпоинт (768px): десктоп не
затронут. БД/Edge не трогались.

### Панель и высота
- `.side-panel`: `height: 100dvh` (фолбэк `100vh`) — динамический viewport
  равен реально видимой высоте, композер больше не уходит под адресную строку.
- `@media (max-width: 767px)`: панель чата/задачи → `width: 100vw` (fullscreen,
  без торчащих краёв); inline `--panel-width` перебит `!important`.
- Ресайз-ручки панели и сайдбара скрыты на мобиле (`hidden md:block`).

### «Прочитано» над композером
- Ряд переверстан с `absolute left-1/2` на flex с зеркальными распорками: левая
  группа (режимы + @) и правый пустой спейсер оба `flex-1`, кнопка между ними
  `shrink-0`. Равные распорки центрируют кнопку, пока хватает места; когда режимы
  шире своей доли — толкают кнопку вправо. Истинный центр без наслоения, без JS.

### Левый сайдбар (мобильный drawer)
- `@media (max-width: 767px)`: `[data-workspace-sidebar]` → `width: calc(100vw -
  44px)` — шире, чем на десктопе, но справа остаётся полоска затемнённого фона
  (видна граница drawer'а, тап по ней закрывает).
- icon-кнопки навигации и быстрых действий выше на мобиле (`h-10 md:h-[30px]`).

### Шапка чата
- Меню «⋮» (`TaskActionsMenu` в `TaskPanelTaskHeader`):
  `opacity-100 md:opacity-0 md:group-hover` — видно на тач-экране, ховер-логика
  на десктопе сохранена.
- Подписи чипов «Срок» (`DeadlinePopover`, только плейсхолдер — дата остаётся) и
  «Проект» (`ChatSettingsProjectSelector`, новый проп `labelClassName`) скрыты на
  мобиле (`hidden md:inline`) — иконки достаточно.

### Верхние вкладки и прокрутка
- `BoardsPage`: верхние строки (раздел + полоса вкладок) получают левый отступ
  под бургер (`pl-14 md:pl-3`); «+» добавить доску виден на тач-экране.
- Утилита `.scroll-fade-right` (CSS-маска): правый край горизонтального списка
  подтаивает как подсказка о прокрутке. Адаптивна — пока контент влезает (списки
  выровнены влево), маска приходится на пустоту и невидима; при переполнении
  тает последний элемент. Применена к полосе вкладок досок и сегментам
  «Входящих».

## Проверки

tsc 0, lint 0, 735 тестов. Десктоп не затронут (всё через `md:`). Живой смок на
телефоне — за пользователем после деплоя: поле ввода видно; «Прочитано» по
центру/сдвинуто без наслоения; панель на весь экран; сайдбар шире с полоской

подтаивает при прокрутке.

## На будущее

- `100vh` в любом fixed-fullscreen контейнере на мобиле — баг (хром браузера),
  использовать `100dvh`.
- Hover-only контролы (`opacity-0 group-hover`) недоступны на тач — раскрывать
  через `opacity-100 md:opacity-0 md:group-hover`.
- Для «центр-или-сдвиг» в ряду — flex-1 распорки по обе стороны, не
  `absolute left-1/2`.
- Подсказка о горизонтальной прокрутке без JS — `.scroll-fade-right` на
  контейнере с `overflow-x-auto`.
