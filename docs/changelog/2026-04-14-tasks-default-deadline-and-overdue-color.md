# Дефолт «без срока» скрыт в «Мои активные» и красное имя у просроченных

**Дата:** 2026-04-14
**Тип:** feat
**Статус:** completed

---

## Проблема

В разделе «Задачи» (`/workspaces/[id]/tasks`) на пресете «Мои активные» в общий список попадали задачи без срока — десятки строк «Без срока» отодвигали вниз действительно срочные задачи. Чтобы убрать их, приходилось каждый раз вручную открывать фильтр «Срок» и снимать чекбокс «Без срока».

Дополнительно: просроченные задачи визуально не отличались от остальных — название было обычным чёрным, а единственным маркером служила маленькая красная плашка с датой справа. На скриншотах пользователь видел просрочку, только дочитав строку до конца.

## Решение

### 1. Эффективный фильтр по сроку (по аналогии со статусами)

`deadlineFilter` в `useTaskFilters` стал `Set | null`, где `null` означает «использовать дефолт пресета». Добавлен `effectiveDeadlineFilter`:

- `my_active` → `{ overdue, today, tomorrow, this_week, later }` (всё кроме `no_deadline`)
- остальные пресеты → пустой `Set` (показывать всё)

Это тот же паттерн, что уже используется для `effectiveStatusFilter` / `statusFilterIds`. Чекбоксы в попапе фильтра отражают именно «эффективное» значение, поэтому пользователь сразу видит, что включено: на «Мои активные» все галочки стоят, а «Без срока» — снят. Можно вручную включить — это сохраняется до смены пресета (там сбрасывается обратно в `null`).

### 2. Красное название у просроченных задач

В `TaskRow` добавлен флаг `isOverdue = !isFinal && getDeadlineGroup(task.deadline) === 'overdue'`. Когда он true:

- к названию применяется класс `text-red-600`;
- inline-`text_color` из статуса **не** применяется — просрочка важнее визуальной стилизации статуса;
- финальные статусы (закрытые задачи) не подсвечиваются, даже если срок давно прошёл.

## Затронутые файлы

| Файл | Изменение |
|------|-----------|
| `src/components/tasks/useTaskFilters.ts` | `deadlineFilter` стал `Set \| null`; добавлен `effectiveDeadlineFilter` с дефолтом для `my_active`; фильтрация и зависимости `useMemo` переключены на эффективное значение. |
| `src/components/tasks/TaskListControls.tsx` | `DeadlineFilter` теперь рендерится по `effectiveDeadlineFilter`; toggle берёт базу `deadlineFilter ?? effectiveDeadlineFilter`. |
| `src/components/tasks/TaskListView.tsx` | Пустое состояние ориентируется на `effectiveDeadlineFilter`. |
| `src/components/tasks/TaskRow.tsx` | Импорт `getDeadlineGroup`; вычисление `isOverdue`; класс `text-red-600` на названии, при этом inline-`text_color` статуса игнорируется при просрочке. |
