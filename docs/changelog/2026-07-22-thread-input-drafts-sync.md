# 2026-07-22 — Черновики поля ввода синхронизируются между устройствами

## Что было

Недописанный текст и прикреплённые файлы жили ТОЛЬКО в браузере (localStorage +
IndexedDB) — на другом устройстве их не было. И список «Входящие» приходит с
сервера, поэтому в принципе не мог знать о начатых черновиках.

## Что стало

1. **Текст** — таблица `thread_input_drafts` (thread_id + user_id, RLS own-rows).
2. **Файлы** — `thread_input_draft_files` (ссылка на уже загруженный `files.id`).
   Прикреплённый файл сразу уходит в Storage; на другом устройстве черновик
   открывается с вложениями.
3. **Вкладка «Непрочитанные»** дополнительно показывает треды с начатым
   черновиком (`get_inbox_unread_threads_impl` — UNION по обеим таблицам,
   фильтры: свои черновики, этот воркспейс, не удалённые треды).
4. **Пометка «Черновик: …»** в строке списка берётся с сервера (видна на любом
   устройстве), локальная версия приоритетнее — она свежее.

## Ключевые решения

**Почему НЕ `project_messages.is_draft`.** На той таблице 15 триггеров, часть — на
любое изменение: пересчёт непрочитанного у всех участников, пересчёт inbox-меты,
realtime-рассылка всему воркспейсу, подъём треда наверх. Автосохранение по ходу
набора дёргало бы всё это на каждую паузу в печати — у коллег тред прыгал бы в
начало «Входящих», пока ты пишешь.

**Local-first.** localStorage/IndexedDB пишутся как раньше (мгновенно, офлайн),
сервер — слой синхронизации (текст с debounce 2с). Иначе при плохой связи
теряются последние набранные слова, а ввод подтормаживает. Конфликт «печатал на
двух устройствах» решается по времени правки (`msg_draft_ts:<threadId>`).

**Путь отправки не тронут.** Наверх уходят обычные `File`, отправка работает как
раньше. Цена — файл, прикреплённый на одном устройстве и отправленный с другого,
загрузится дважды. Осознанный размен: отправка вложений — самое инцидентоопасное
место проекта.

**Пометка и «смешанный» бейдж — отдельным запросом**, а не полем в резолвере
инбокса: там 35 колонок и 7 зависимых функций, добавление поля тянет DROP+CREATE
всей цепочки с восстановлением грантов.

## Мусор в хранилище

Отправка и удаление файла чистят серверную копию вместе с объектом Storage
(отправленное сообщение загружает собственные копии). Брошенные черновики
хранятся бессрочно — решение владельца.

## Осознанно не сделано

- Существующие localStorage-черновики не переносятся на сервер (уедут, когда в
  треде снова начнут печатать).
- Счётчик на вкладке «Непрочитанные» черновики НЕ считает — по решению владельца.
- Файлы догружаются с сервера, только если локально их нет (иначе пришлось бы
  сливать два набора).
