# Источники Google Drive: набор из папки, лоток файлов, унификация источников

**Дата:** 2026-07-09
**Тип:** feat (документы; фронт + edge `google-drive-*` + БД миграции)
**Статус:** БД-миграции и edge-функции задеплоены; фронт — деплой push в main → CI/CD blue/green

---

Большая фича вокруг подключения Google Drive к наборам документов. Свели два
раздельных механизма (набор документов ↔ источник в правой панели) в единую
модель источников. Разрабатывалось маленькими шагами; здесь — сводка.

## 1. Создание набора документов из папки Google Drive

Новый способ создать набор — по ссылке на папку Drive (пункт меню «···» вкладки
«Документы» → «Из папки Google Drive»). Корневая папка → набор, подпапки первого
уровня → папки набора, файлы клиента зеркалятся в набор.

- Edge `google-drive-list-folders` (новая) — имя корня + подпапки первого уровня.
- Edge `google-drive-list-files` — режим `groupByTopLevel`: файл относится к
  подпапке ПЕРВОГО уровня (по имени и id), даже если лежит во вложенной; корневые
  файлы — с пустой папкой.
- Сервис `documentKitDrive.ts` → `createDocumentKitFromDriveFolder`.

## 2. Единая связь с Drive-папкой (`drive_folder_id`)

Вместо двух полей-близнецов (`source_folder_id` + `drive_folder_id`) у набора и
папок — одно поле `drive_folder_id` (связь с Drive-папкой), заполняется и при
создании из Drive, и мастером «Создать папки на Google Drive».

- Файлы источника сопоставляются с папкой набора по **ID Drive-подпапки первого
  уровня** (`source_documents.parent_drive_folder_id`, миграция
  `20260709130000`) — устойчиво к вложенности, дефисам и переименованию.
- Вкладка «Внешние» (share-links) показывает папки набора автоматически через
  существующие `kit_folder`/`doc_folder` — отдельные ветки-источники не нужны.

## 3. Лоток файлов источника внутри набора

Под сохранёнными документами в папке набора — «лоток» файлов из Drive
(компонент `KitSourceFileRow`):

- размер и дата загрузки у каждого файла;
- кнопки на hover: **«Открыть»** (в Drive), **«Скрыть»** (глаз, флаг `is_hidden`),
  **«Принять»** (скачать из Drive и добавить документом в папку — переиспользует
  тот же путь, что перетаскивание);
- корневые файлы Drive-папки — отдельным блоком на уровне набора;
- пункт меню набора «Обновить файлы из источника».

## 4. Фильтр документов вместо переключателя

Переключатель «Все / Требуется действие» + чекбокс «Показывать скрытые файлы»
убраны под кнопку **«Фильтр»** (раскрывающаяся строка, задел под будущие фильтры).
Оранжевая точка на кнопке = фильтр применён.

## 5. Унификация источников (таблица `document_sources`)

Миграция `20260709140000` — таблица `document_sources` (id, project_id,
workspace_id, `document_kit_id` nullable, `drive_folder_id`, name). Источник =
папка Drive; наборные (привязка к набору) и отдельные (null) — в едином списке.
Файлы ссылаются на источник через `source_documents.source_id`. Перенос данных:
82 источника, ~1962 файла привязаны (аддитивно, старые поля целы).

Правая панель «Из источника» теперь:

- показывает **ВСЕ** источники, сгруппированные **источник → папка → файлы**
  (заголовок источника — имя набора / «Источник проекта»);
- **«＋ Добавить источник»** — по ссылке Drive (создаёт `document_sources` +
  синк);
- **корзина** на заголовке — удалить отдельный источник (не наборный) с файлами;
- **синхронизация** ↻ на заголовке каждого источника (наборный — с группировкой
  по папкам первого уровня, отдельный — по ближайшей папке).

## 6. Синхронное скрытие панель ↔ лоток

Скрытие файла (флаг `is_hidden`) синхронно во всех местах: скрыл в панели —
пропал и в лотке набора, и наоборот. Единый сброс кэшей
`useInvalidateAllSourceViews` (панель + лотки) на любом изменении файла источника
(скрытие, приём, синхронизация).

## Грабли / заметки

- **Наборные файлы видны в двух местах** (панель + лоток набора) — by design
  (обзор vs фокус); дубля данных нет, файл принадлежит одному источнику.
- **Маппинг `source_documents` → `SourceDocument`** есть в двух местах
  (`fetchSourceDocuments` для панели и `fetchKitSourceDocuments` для лотка) —
  новые поля (`source_id`, `parent_drive_folder_id`) добавлять в оба, иначе
  панель/лоток теряют привязку (был баг «Без источника» при заполненном `source_id`).
- **Синхронизация источника** в трёх точках: заголовок источника в панели, меню
  набора, старая кнопка шапки секции (legacy проектный источник).

## БД / edge (задеплоено)

- Миграции: `20260709130000_source_documents_parent_drive_folder_id.sql`,

- Edge: `google-drive-list-folders` (новая), `google-drive-list-files`
