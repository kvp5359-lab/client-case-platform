# Листаемые ряды вкладок + верстка «Обновлений источников» под боковую панель

**Дата:** 2026-07-13
**Тип:** UX (фронт: вкладки проекта/досок/панели + страница «Обновления источников»)
**Статус:** деплой (push в main → CI/CD blue/green)

---

Чисто фронтовая сессия по доработке поведения интерфейса. Мессенджера/БД/edge не
касалась. Проверка фикса скролла велась замером в браузере (перехват
`scrollIntoView`), а не по чтению кода.

## 1. «Обновления источников»: верстка под открытую боковую панель

**Симптом.** При открытой правой панели контент страницы `/source-updates` уезжал
под панель (центрированный `max-w-4xl mx-auto` не учитывал панель).

**Решение.** Страница переведена в push-режим панели — как «Доски», «Задачи»,
«Финансы», «Документы»: `document.body.setAttribute('data-panel-mode', 'push')` в
эффекте (cleanup снимает атрибут). CSS-правило
`body[data-panel-open][data-panel-mode="push"] main { margin-right: var(--panel-width) }`
уже существует → `main` отжимается влево, лента помещается в свободное место.

**Заодно.** Описание страницы («Файлы из привязанных папок…») зажималось слева
между заголовком и кнопками (одна flex-строка). Перекомпоновано: строка
«заголовок ↔ кнопки», а описание — отдельным абзацем на всю ширину колонки ниже.

## 2. Ряды вкладок: листание скроллом вместо overflow-в-бутерброд

**Запрос.** Вкладки должны листаться горизонтально (свайп/скролл), но справа
всегда оставаться кнопка-«бутерброд»; видимого ползунка прокрутки не нужно.

**Было.** Общий хук `useOverflowTabs` мерил ширины в скрытом ряду-клоне и прятал
непоместившиеся вкладки в бутерброд (без скролла). Использовался в двух местах —
вкладки модулей проекта (`ProjectModuleTabBar`) и вкладки досок/списков
(`BoardsPage`).

**Стало.** Ряд — горизонтальный скролл-контейнер `overflow-x-auto scrollbar-hide`
(ползунок скрыт), все вкладки рендерятся; бутерброд вынесен ВНЕ скролла (всегда
справа), в его меню — все вкладки, активная подсвечена. Активная вкладка при
переключении подскролливается в зону видимости. Паттерн взят из уже работающего
`TaskPanelTabBar` (панель вкладок треда).

- `useOverflowTabs` удалён (клон-ряд + `ResizeObserver` + арифметика замеров
  больше не нужны).
- Логика «скролл к активной вкладке» вынесена в единый хук
  `useScrollActiveTabIntoView(containerRef, activeId, opts?)` и подключена во всех
  ТРЁХ местах (проект / доски / панель треда) — раньше это была тройная копипаста
  с уже разошедшимися деталями (`useEffect` vs `useLayoutEffect`, `CSS.escape`).
  `TaskPanelTabBar` сохранил свои `behavior: 'smooth'` и `deps: [orderedTabs]`
  через опции хука.

## 3. Фикс: выбор вкладки через бутерброд не подскролливал к ней

**Симптом.** При выборе скрытой (за краем) вкладки через бутерброд она становилась
активной, но ряд к ней не подскролливал — оставалась вне видимости.

**Диагностика (замер в браузере, не гадание).** Перехват `scrollIntoView`:
ручной вызов на активной вкладке работал (`scrollLeft` 25→602), значит механизм
рабочий. Синхронный скролл в `useLayoutEffect` терялся в гонке — при закрытии
Radix-`DropdownMenu` восстановление фокуса/раскладки происходит уже ПОСЛЕ нашего
скролла и сбивает позицию.

**Решение.** Скролл в хуке идёт в два прохода: синхронно (обычный случай) +
повторно в `requestAnimationFrame` (перекрывает сброс после закрытия меню).
Cleanup отменяет rAF.

**Проверка вживую.** Доски, узкий ряд, выбор скрытой вкладки «Все» через бутерброд:
перехват показал `scrollIntoView` 0→485, заголовок сменился на «Все», активная
вкладка видна в ряду у правого края. Раньше `scrollLeft` оставался прежним и
вкладка была за кадром.

## Прочие мелочи верстки

- Проектные вкладки: убран `gap-1` у скролл-ряда (вкладки снова вплотную, как было
  до правки; на досках `gap-1` был исходно — оставлен).
- Скролл-контейнерам добавлен вертикальный припуск `py-0.5 -my-0.5`, чтобы
  `overflow` не подрезал тень активной вкладки (высота ряда не меняется).
- `SourceUpdatesPage`: переотбит отступ блока кнопок после переноса.
