## 1. Префикс названия проекта — отображаемое свойство шаблона (feat)

### Проблема / задача
У шаблона проекта было поле «Название по умолчанию» (`default_name_prefix`),
которое при создании проекта **вшивалось в само имя** (`projects.name` =
«Лид: Иванов»). Это мешало: имя несло префикс везде (заголовки, поиск), а
управлять им отдельно было нельзя. Нужно: префикс — отдельное свойство шаблона,
показываемое опционально (сайдбар, шапка проекта, поле создания), не меняя имя.

### Решение (вариант «отвязать префикс от имени»)
- **БД (прод):** колонка `project_templates.show_name_prefix_in_sidebar boolean
  NOT NULL DEFAULT false`. Миграция `20260630_project_template_show_name_prefix_in_sidebar.sql`.
- **Создание проекта:** префикс больше НЕ склеивается в `projects.name` —
  имя создаётся чистым. Если шаблон разрешил показ — префикс рисуется серым
  **внутри поля названия** (нередактируемый, имя печатается после него).
- **Сайдбар:** строка проекта рисует `<префикс> <имя>` (префикс приглушённым
  цветом), когда у шаблона включён флаг. Запрос шаблонов в `useSidebarData`
  дополнен `default_name_prefix` + флагом; enrichment кладёт `namePrefix` в проект.
- **Шапка проекта:** префикс серым слева от имени (в обоих режимах — просмотр и
  inline-редактирование; имя редактируется без префикса).
- **Редактор шаблона:** лейбл «Название по умолчанию» → «Префикс названия
  проекта»; рядом чекбокс «Отображать префикс» (один флаг управляет всеми тремя
  местами: сайдбар + шапка + создание; выключен → префикс нигде).

### Грабли (важно)
- Сайдбар тянет шаблоны **отдельным** запросом `sidebarMetaKeys.templatesIcons`
  (НЕ под ключом редактора). Мутации префикса/флага обязаны инвалидировать
  `sidebarMetaKeys.templatesIconsAll` (broad-prefix), иначе сайдбар не обновится
  без перезагрузки.
- Префикс — отображаемое свойство шаблона, в `projects.name` его больше нет.
  Старые проекты, у которых префикс был вшит в имя, миграцией данных не трогались
  отдельной задачей (см. ниже разовую чистку лидов).

### Разовая чистка данных (прод)
У лидов префикс исторически был вшит в `projects.name` («Лид Ю: …», «Лид ВНЖ: …»).
После включения флага получался двойной префикс. Разовым UPDATE срезан вшитый
префикс у **108** проектов (Лид ЮРИСТ — 106, Лид ВНЖ — 2). `Лид DN: Vanya` не
тронут (другой префикс). Пустых имён после обрезки нет.

### Фикс гонки общего кэша (префикс не показывался в сайдбаре)
Сайдбар (`useSidebarData`) и поиск (`useProjectTemplateIcons` в `useGlobalSearch`)
делят ОДИН кэш-ключ `sidebarMetaKeys.templatesIcons`, но имели РАЗНЫЕ queryFn:
у поиска select был без `default_name_prefix`/`show_name_prefix_in_sidebar` и map
без `namePrefix`. При монтировании обоих хуков React Query при первом fetch
недетерминированно выбирал queryFn — часто выигрывал «старый» (поиска) → кэш без
`namePrefix` → префикс в сайдбаре не появлялся (шапка работала, у неё свой ключ
`detail`). Фикс: queryFn `useProjectTemplateIcons` выровнен с сайдбаром (тот же
select + `namePrefix` в map + поле в `ProjectTemplateMeta`).
**Грабли:** два хука на одном queryKey ОБЯЗАНЫ иметь эквивалентный queryFn —
иначе содержимое общего кэша зависит от того, кто выиграл гонку первого fetch.

## 2. Статус задачи в композере показывает имя выбранного статуса (fix)

`TaskStatusPicker` в композере всегда показывал хардкод «Новый статус», даже
после выбора статуса. Текст заменён на имя выбранного статуса (`displayed?.name`);
«Новый статус» остаётся только когда ничего не выбрано.

**Файл:** `src/components/messenger/TaskStatusPicker.tsx`
