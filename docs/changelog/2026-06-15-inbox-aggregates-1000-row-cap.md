# Входящие: агрегаты обрезались на 1000 строк → непрочитанный тред «хвоста» без бейджа и с кнопкой «Непрочитано»

**Дата:** 2026-06-15
**Тип:** fix
**Статус:** completed (БД в проде; фронт ждёт деплоя)

## Симптом

У непрочитанной задачи в боковой панели кнопка показывала «Непрочитано»
(должна «Прочитано»), на вкладке панели не было бейджа непрочитанного. При этом
в списке «Непрочитанные» у того же треда бейдж был. Жёсткая перезагрузка не
помогала; воспроизводилось и на проде, и локально.

## Корень

**Supabase REST отдаёт максимум 1000 строк за один RPC-запрос** (лимит
инфраструктуры PostgREST; в `pg_settings` / `db_max_rows` не виден). Лёгкий
запрос `get_inbox_thread_aggregates` грузил ВСЕ доступные треды одним вызовом
`supabase.rpc(...)` без пагинации.

Воркспейс перевалил 1000 тредов (1122 всего, 1099 доступных) → клиент получал
ровно первые 1000, а треды «хвоста» молча выпадали. Всё, что построено на
агрегатах — кнопка «Прочитано/Непрочитано», бейджи вкладок панели, бейдж проекта
в сайдбаре, счётчики сегментов «Нужно ответить»/«Ждём клиента» — для треда вне
1000 видело 0/false. Список «Непрочитанные» работал, т.к. это отдельный RPC,
объём которого 1000 не достигает.

Латентный баг с момента, как воркспейс перешагнул 1000 тредов.

### Как нашли

Замер рантайма: чтение кэша React Query через React fiber → `queryClient`.
Кэш `inboxKeys.aggregates` содержал ровно 1000 строк, искомого треда в нём не
было; `inboxKeys.unread` — 7 строк, тред присутствовал. Сервер при этом отдавал
1099 строк (проверено отдельно). Разрыв — на транспорте.

## Фикс

- **БД** `get_inbox_thread_aggregates` + `ORDER BY at.id` — детерминированный
  порядок, обязательный для стабильной постраничной выборки. `CREATE OR REPLACE`,
  тип возврата не менялся → гранты сохранены (`authenticated`, `service_role`).
- **Фронт** `getInboxThreadAggregates` тянет данные постранично через
  `.range(from, from + 999)` в цикле, пока страница полная (1000) — тот же
  паттерн, что `boardFilterService.getBoardFilteredThreads` для досок.

Проверено в preview: кэш агрегатов стал полным (1100 строк), тред присутствует,
кнопка → красная «Прочитано», бейдж вкладки появился.

## На будущее (G9 в messenger-ledger)

Любой inbox-RPC, грузящий весь набор тредов одним `supabase.rpc()` без `.range()`,
обрежется на 1000. Ещё не пагинированы (ок пока < 1000, но растут):
`get_inbox_unread_threads`, `get_inbox_needs_reply_threads` (~322),
`get_inbox_awaiting_reply_threads`, `get_inbox_search_threads`. Пагинировать при
приближении к 1000 (с `ORDER BY` в RPC).
