# Title вкладки браузера сохраняется при смене вкладки проекта

**Дата:** 2026-04-29
**Тип:** fix
**Статус:** completed

---

## Контекст

При переходе между вкладками проекта (Задачи → Документы → Настройки) title вкладки браузера сбрасывался на дефолтный «ClientCase». Раньше показывалось имя проекта (`Мария Старостина — ClientCase`), а после клика по «Настройки» становилось просто `ClientCase`. В Chrome с боковой панелью все три открытые вкладки проекта стали выглядеть одинаково — невозможно отличить.

## Причина

Next.js App Router пересчитывает metadata на каждой навигации, включая `router.replace` с новым query-параметром (например, `?tab=settings`). При пересчёте `document.title` затирается дефолтным значением из root layout (`'ClientCase'`).

`usePageTitle` запускал свой эффект только при смене значения `title`. Когда менялся только URL, а title оставался прежним (тот же проект, та же страница), эффект не срабатывал — наш title не перевыставлялся, в DOM оставался дефолт от Next.js.

## Решение

В `usePageTitle` добавлены зависимости `pathname` и `searchParams` (из `next/navigation`). Любая навигация триггерит эффект — он перевыставляет правильный title после того, как Next.js затёр его metadata-обновлением.

В `ProjectPage` title собран из имени проекта и названия активной вкладки: `Мария Старостина — Настройки`. Метку вкладки берём из `availableModules` (поле `label` в `moduleRegistry`).

## Файлы

- `src/hooks/usePageTitle.ts` — `pathname` + `searchParams` в deps эффекта
- `src/page-components/ProjectPage.tsx` — составной title `<проект> — <вкладка>`

## Почему так

**Зависимости URL внутри хука, а не через явный bump-флаг.** Альтернатива — в каждом потребителе хука кидать `key` на основе pathname, или возвращать функцию `refresh()`. Это потребовало бы трогать каждое место использования. Хук сам понимает «URL изменился» — это inline по месту, не плодит boilerplate.

**Title с именем вкладки, а не только проекта.** Без вкладки одинаковый title во всех табах Chrome у пользователя с тремя открытыми вкладками одного проекта — главный пейн-пойнт, ради которого правка вообще делалась.
