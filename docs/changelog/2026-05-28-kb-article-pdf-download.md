# База знаний — скачивание статьи в PDF из вкладки тред-панели

**Дата:** 2026-05-28
**Тип:** feature
**Статус:** completed

---

## Контекст

Сотрудники часто хотят сохранить статью базы знаний оффлайн (например,
взять на встречу с клиентом или переслать). До этого приходилось через
браузерный «Print → Save as PDF», что давало результат с лишними
элементами интерфейса и неудобными стилями.

---

## Что сделано

Добавлена кнопка «Скачать PDF» (иконка `Download` из lucide) в шапке
вкладки [`KnowledgeArticleTabContent`](../../src/components/tasks/KnowledgeArticleTabContent.tsx)
— рядом с крестиком закрытия. По клику открывается системный диалог
сохранения PDF.

### Технически

Утилита [`printArticleToPdf`](../../src/utils/print/printArticle.ts):

- Создаёт **скрытый iframe**, в него копирует HTML с контентом статьи.
- Накладывает inline-стили (`@page margin`, prose-настройки) — шрифты
  заголовков, отступы, кода, блок-цитат, таблиц. PDF-friendly, без
  оглавления UI.
- Клонирует DOM (не innerHTML) — сохраняет **текущее состояние
  раскрытых `<details>` аккордеонов**. Если юзер раскрыл секцию перед
  скачиванием — она будет раскрыта и в PDF.
- Вызывает `iframe.contentWindow.print()` — браузер показывает диалог
  «Save as PDF», название по умолчанию = title статьи.
- После закрытия диалога — iframe удаляется.

### Видимость кнопки

Контролируется тем же критерием, что и доступ к Ctrl+C/Ctrl+A:

```ts
const canDownloadPdf = !!article && (!isReadOnly || isOwner)
```

То есть:
- Если статья **не read-only** → кнопка видна всем.
- Если статья **read-only** (защищённая от копирования) → только
  владельцу воркспейса.

Это уважает защиту от слива контента клиентам / внешним юзерам, но не
блокирует владельца, для которого PDF-выгрузка — легитимный сценарий.
