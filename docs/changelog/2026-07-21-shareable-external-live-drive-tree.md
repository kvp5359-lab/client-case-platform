# 2026-07-21 — Вкладка «Внешние» пикера молнии: живое дерево Google Drive

## Проблема

Вкладка «Внешние» пикера молнии строила дерево ссылок из **связей в БД**
(`get_project_shareable_resources.external`), которые могут расходиться с реальной
структурой папок проекта на Google Диске (папки, созданные/переименованные на
Диске напрямую). Плюс в «Внешних» показывалась «Анкета» (`form_kits.google_sheet_id`),
хотя в рабочем процессе набор генерит только бриф — лишняя, путающая ссылка.

## Что сделано

1. **Edge `google-drive-shareable-tree`** (нов) — читает **живую** структуру
   папок проекта прямо с Google Drive: рекурсивно обходит папки под корневой
   папкой проекта (до `MAX_DEPTH=6`, `MAX_FOLDERS=300`), находит реальную папку
   брифа. Возвращает плоский список узлов `{kind, id, parent_id, label, url}` —
   дерево фронт собирает по `parent_id`. Файлы (кроме брифа) не листает. Токен
   Drive — через `ensureValidAccessToken`, доступ — `checkWorkspaceMembership`.
2. **Фронт** — «Внешние» рисуют живое Drive-дерево (`getProjectDriveExternalTree`
   → edge); если Диск не подключён / не прочитался — фолбэк на дерево из БД
   (`normalizeDbExternal` приводит `data.external` к тому же виду `id/parent_id`).
   Рендер — `ExternalLinksView` по `parent_id` с линиями-гайдами базы знаний
   (`TreeConstants`).
3. **RPC `get_project_shareable_resources`** (миграция `20260721120000`) — убрана
   ветка «Анкета» (`google_sheet_id`) из внешних ссылок; остаётся только бриф
   (`brief_sheet_id`). Данные не трогаются, только перестают показываться.
   Резолв эффективной статьи слота (`coalesce`, фикс молнии от 2026-07-18)
   сохранён.
4. **Инвалидация кэша** — новый `projectShareableKeys` (`byProject` + подключ
   `driveTree`). Любая мутация, меняющая состав «Внешних», сбрасывает его:
   смена корневой папки проекта (`useProjectMutations`), генерация/пересоздание
   брифа (`useBriefSheetActions`), привязка набора к Drive-папке
   (`useDocumentKitsQuery`), создание Drive-папки в мастере (`useDriveFoldersWizard`
   — новая `linkDriveFolderToKit`, пишет `document_kits.drive_folder_id` сразу,
   не дожидаясь шага «Подпапки»).

## Грабли

- «Внешние» = живое Drive-дерево при подключённом Диске, иначе фолбэк на БД
  (менее точный). Оба вида приводятся к `id/parent_id` для единого рендера.
- Любая мутация, меняющая внешние ссылки проекта, обязана инвалидировать
  `projectShareableKeys.byProject` — иначе пикер покажет старое до перезагрузки.
- Миграция дропа «Анкеты» сохраняет `coalesce`-резолв статьи слота — при правке
  этой RPC держать обе вещи (иначе откат фикса молнии).
