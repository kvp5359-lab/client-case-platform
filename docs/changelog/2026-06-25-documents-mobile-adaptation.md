# Мобильная адаптация модуля «Документы»

**Дата:** 2026-06-25
**Тип:** fix/feat
**Статус:** completed (ждёт деплоя фронта)

---

Часть общей мобильной адаптации (см. `2026-06-25-mobile-adaptation*.md`). Всё —
фронт (CSS/классы), через `md:`-брейкпоинт и `isMobileViewport()`: десктоп не
затронут.

## Проблемы

1. **Строка документа**: открытие-иконка + ⋮ показывались только по ховеру (на
   тач недоступны), а после скрытия иконки оставался большой пустой кусок справа
   от ⋮; пустой бейдж комментариев резервировал место.
2. **Имя папки пропадало полностью** на узком экране — скрытые ховер-кнопки
   «Слот»/«Документы» занимали место, имя (`min-w-0 truncate`) ужималось до нуля.
3. **Общая правая пустота** контента документов на мобиле.

## Что сделано

### Строка документа (`DocumentItem.tsx`)
- **⋮ всегда видно** на мобиле (`opacity-100 md:opacity-0 md:group-hover`).
- **Иконка открытия скрыта** на мобиле (`hidden md:flex`) — открытие через меню.
- **Тап по названию → открывает меню «⋮»** (`isMobileViewport()` → `setMenuOpen(true)`,
  `stopPropagation`); на десктопе клик по строке правит слот как раньше.
- **Имя `flex-1`** — растягивается на всё свободное место, размер/бейдж/⋮ уезжают
  к правому краю (убрана пустота справа от ⋮, максимум места под название).
- **Пустой бейдж комментариев скрыт** на мобиле (`hidden md:inline-flex`) — был
  `opacity-0`, занимал место. С комментариями — показывается как обычно.

### `DocumentActionsMenu.tsx`
- Добавлен проп `open` (контролируемое открытие) — для вызова меню по тапу
  на название. `undefined` → прежний неконтролируемый режим.

### Шапка папки (`FolderCardHeader.tsx`)
- На мобиле кнопки «Слот»/«Документы» (ховер-скрытые, занимавшие место) собраны
  в **одну видимую «+»** с меню (Загрузить документы / Добавить слот). Десктоп —
  прежние две кнопки по ховеру. Пустой бейдж комментариев скрыт на мобиле.

### Правые отступы на мобиле (меньше пустоты)
- Контент проекта (`ProjectPage.tsx`): `p-3` → правый `pr-1`.
- Карточка документов (`KitDocuments.tsx`): `pr-2` → `pr-0.5`.
- Контент папки (`FolderCard.tsx`): `pr-2` → `pr-0.5`.
- Шапка папки: `pr-3` → `pr-1`. Строка документа: `pr-2.5` → `pr-1`.
- Все — через `md:` (десктоп без изменений).

## На будущее

- Hover-only контролы (⋮, кнопки добавления) недоступны на тач — раскрывать на
  мобиле (`opacity-100 md:opacity-0 md:group-hover`) или собирать в видимое
  меню «+».
- Чтобы убрать «пустоту справа» в flex-строке — растягивающий элемент `flex-1`
  (имя), остальное `shrink-0`.
- Пустой бейдж комментариев (`emptyClassName`) на мобиле — `hidden`, не
  `opacity-0` (иначе резервирует место).
