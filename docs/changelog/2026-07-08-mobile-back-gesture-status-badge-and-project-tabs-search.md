## A. Мобильная версия — фиксы

### A1. Свайп-назад: «двойное закрытие» панели/чата

**Проблема.** При закрытии боковой панели треда (доски) или чата (инбокс)
**жестом свайпа** на Android визуально казалось, что панель закрывается дважды —
«под панелью видно её же второй раз». При закрытии кнопкой «Назад» в браузере —
нормально. Один жест давал правильный результат (панель закрывается), просто
выглядело как двойное закрытие.

**Корень.** Открытие треда/чата добавляет запись в историю браузера
(`router.push` / `history.pushState`). Закрытие по «Назад» → `popstate` → наша
панель проигрывает слайд-выезд 300мс. При **свайпе** Chrome дополнительно рисует
свою **нативную анимацию перехода** (скриншот предыдущего состояния страницы —
той же SPA-страницы с открытой панелью). Два слоя — нативный скриншот Chrome +
наш DOM-слайд — накладывались → «двойная панель». Кнопка «Назад» такой
скриншот-анимации не рисует → там чисто.

**Решение.** Механизм истории/«Назад» не тронут (он и есть то, что делает
кнопку/жест рабочими). Меняется только анимация закрытия по правилу «кто
инициировал»:

| Как закрыли | Анимация |
|---|---|
| Свайп / кнопка «Назад» браузера (`popstate`) | **мгновенно**, без нашего слайда |
| Крестик / «← Входящие» в приложении | слайд-выезд 300мс, как раньше |

Так на свайпе остаётся только нативная анимация Chrome (нет наложения), на
кнопке «Назад» — мгновенно (и так выглядело чисто), а крестик — плавно.

**Как отличаем.** В `popstate` нельзя понять «жест это или кнопка», но это и не
нужно: разница в том, что крестик закрытие инициируем МЫ, а «Назад» — браузер.
- **Доски** (`TaskPanelTabbedShell` + рендерер): ref `closeAnimateRef` (по
  умолчанию «анимировать»); в ветке закрытия по `popstate` ставим «мгновенно».
  Рендерер читает ref в момент закрытия: мгновенно → `setRender(false)` сразу,
  без 300мс-таймера и слайда; затем сбрасывает ref в true.
- **Инбокс** (`InboxPage`): крестик/«← Входящие» перед `history.back()` ставит
  флажок `chatCloseAnimateRef` → `popstate` понимает, что закрытие «наше»
  (анимируем). Свайп/«Назад» → флажок не выставлен → `chatInstantClose`: гасим
  `transition` у панели и сразу убираем контент чата. На открытии чата флажок
  сбрасывается — вход остаётся со слайдом.

**Грабли.** Механизм «Назад» на досках — многовкладочный (каждое открытие треда
= запись истории), поэтому крестик там НЕ переводили на `history.back()` (он
закрывает всю панель, а `history.back()` вернул бы предыдущую вкладку). В инбоксе
крестик уже шёл через `history.back()` — там добавили только различение анимации.

**Файлы.** `TaskPanelTabbedShell.tsx`, `TaskPanelTabbedShellRenderer.tsx`,
`InboxPage/index.tsx`.

### A2. Статус проекта давил название в шапке панели (мобила)

**Проблема.** В верхней строке боковой панели проекта (`PanelProjectInfoRow`)
чип статуса не сжимался вообще, из-за чего название проекта обрезалось почти
полностью.

**Решение.** `ProjectStatusPopover` получил проп `triggerClassName` (по умолчанию
поведение прежнее) + название статуса обёрнуто в `truncate`. Шапка передаёт
`triggerClassName="shrink min-w-0 md:shrink-0"` — на мобиле статус сжимается и
усекается наравне с названием проекта и шаблоном, на десктопе (md+) не трогается.

**Файлы.** `ProjectStatusPopover.tsx`, `PanelProjectInfoRow.tsx`.

### A3. Крестик вкладки поверх бейджа непрочитанного (мобила)

**Проблема.** На вкладке с непрочитанными крестик закрытия накладывался поверх
бейджа непрочитанности.

**Корень.** На мобиле крестик всегда видим (`md:opacity-0 md:group-hover` → без
`md` = `opacity-100`), а бейдж не прячется (нет hover) → крестик ложится поверх.

**Решение.** Зарезервировал справа место под всегда-видимый крестик:
`pl-2 pr-6 md:pr-2` у неактивных (не-pinned) вкладок. Бейдж (в потоке) встаёт
левее крестика. На десктопе — `pr-2` как раньше (крестик по hover перекрывает
угасающий бейдж).

**Файл.** `tab-bar/DraggableTab.tsx`.

## B. Вкладки модулей проекта — скролл вместо пряток

**Было.** При нехватке ширины подписи неактивных вкладок скрывались совсем
(порог container-query `.proj-tab-label--secondary → display:none`), оставалась
иконка.

**Стало.** Подписи показываются ВСЕГДА у всех вкладок: активная — целиком,
неактивные — с равным `max-w-[6.5rem]` (длинные обрезаются многоточием
одинаково). Когда ряду тесно, `TabsList` (`overflow-x-auto scrollbar-auto-hide`)
скроллится по горизонтали — вместо неравномерного ужимания и пряток.

**Файлы.** `globals.css` (убран порог-скрытие), `ProjectPage.tsx`.

## C. Глобальный поиск — префикс проекта + порядок

- Проекты идут **первыми** в результатах поиска (`order` = `['project',
  'thread', ...]`) — в сайдбаре и на странице поиска.
- Перед названием проекта показывается **префикс шаблона** (как в сайдбаре) —
  новый хук `useProjectPrefixResolver` (по `template_id`, учитывает
  `show_name_prefix_in_sidebar`).

**Файлы.** `useGlobalSearch.ts` (+`useProjectPrefixResolver`),
`SidebarGlobalSearch.tsx`, `SearchPage/index.tsx`.
