## 1. Окно поиска+фильтров внутри треда (мессенджер)

Кнопка-лупа в шапке чата теперь открывает полноэкранное окно поиска (оверлей
области чата), а не узкое инлайн-поле.

**Возможности:**
- **Текстовый поиск** по сообщениям треда (как раньше) + **фильтры-типы**
  мультивыбором: `Картинки` / `Аудио` / `Файлы` / `Ссылки`. Пустой запрос +
  фильтр = все элементы этого типа (медиа-вкладка). «Файлы» = документы (не
  картинка и не аудио).
- **Фильтр по отправителю** — селектор «Все отправители» + список авторов треда
  (появляется, если авторов больше одного).
- **Два вида:**
  - **Сообщения** — переиспользуется тот же `MessageList`, что и основная лента:
    бейджи дат («Сегодня/Вчера/дата»), баблы, вложения, аудиоплееры, реакции,
    меню — всё как в чате, ничего не дублируется.
  - **Галерея** — элементы сгруппированы по периоду (Telegram-стиль:
    Сегодня/Вчера/месяц); медиа — сеткой плиток с лайтбоксом, аудио — плеером
    (`AudioAttachmentPlayer`), файлы — через `FileAttachment` (кликом
    открываются), ссылки — списком. У файлов/аудио/ссылок виден отправитель.
- **Строка фильтров** — всё в одну строку: сегмент типов иконками с адаптивными
  подписями (появляются на широкой панели через container query, на узкой —
  только иконки), компактный селектор отправителя, переключатель вида.
  Активный фильтр — мягкий `bg-brand-100 text-brand-600` (как «Все задачи»).

**Сервер (RPC, SECURITY INVOKER — чтение под RLS, канальной логики не касается):**
- `search_thread_messages(thread, query, want_files, want_images, want_links,
  want_audio, sender_participant_id, limit)` — id подходящих сообщений; клиент
  догидрирует полным `MESSAGE_SELECT`.
- `get_thread_senders(thread)` — список отправителей для селектора.

Детали и журнал итераций — [`messenger-ledger.md`](../../.claude/rules/messenger-ledger.md)
(записи 2026-07-10). Миграции: `20260710140000_search_thread_messages`,
`20260710160000_thread_search_sender_filter`, `20260710180000_thread_search_audio_filter`.

## 2. Авто-обновления источников Google Drive

Раньше источники (папки Drive проекта) сканировались только вручную по кнопке.
Добавлена серверная авто-проверка и индикатор непрочитанного.

- **Ежечасный pg_cron** `run_source_sync()` через `net.http_post` (с
  `x-internal-secret`) дёргает новую edge-функцию **`sync-source-documents`**
  (`--no-verify-jwt`). Она обходит все источники живых проектов, на каждую папку
  подбирает токен сотрудника (сначала «кто подключил» — `document_sources.connected_by`,
  затем любой с подключённым Drive; протух — следующий; никто — пропуск), листает
  Drive и upsert'ит в `source_documents`. `created_at` при обновлении НЕ трогается —
  это стабильная метка «файл впервые появился», на ней держится бейдж.
- **Общий обход Drive** вынесен в `_shared/googleDriveFiles.ts` — ручной листинг
  (`google-drive-list-files`) и авто-синк используют одну логику.
- **Бейдж «Обновления источников»** в сайдбаре = число ДОСТУПНЫХ проектов, где
  есть файл, появившийся позже отметки прочтения (`source_update_reads` по
  пользователю и проекту). Пункт форсит этот режим бейджа независимо от
  настройки слота.
- **Скоуп** — обновления считаются/показываются только **исполнителям и
  администраторам** проекта (роль `Исполнитель`/`Администратор` в
  `project_roles`); клиенты, участники и владелец без такой роли не видят.

Миграции: `20260710150000_document_sources_connected_by`,
`20260710160000_source_update_reads`, `20260710170000_source_sync_cron`,
`20260710190000_source_updates_executor_scope`,
`20260710200000_source_updates_include_admins`.

## 3. Единый overflow-механизм рядов вкладок

Новый хук `useOverflowTabs` — общий для вкладок проекта и досок/списков: вкладки,
не помещающиеся по ширине ряда, уходят в кнопку-«бутерброд» справа (без
горизонтального скролла), активная вкладка всегда видима. Ширины меряются в
скрытом ряду-клоне (`measureRef` с `data-tab-id`), `ResizeObserver` пересчитывает
при изменении ширины ряда (в т.ч. при открытии боковой панели). `activeExtraPx` —
доп. ширина активной вкладки (напр. «⋮»-меню, которого нет в клоне), чтобы число
видимых вкладок не скакало при переключении.

Применён в `ProjectModuleTabBar` (вкладки модулей проекта) и `BoardsPage`
(единый ряд досок + списков). Файлы: `useOverflowTabs.ts`, `ProjectModuleTabBar.tsx`,
`BoardsPage/index.tsx`, `BoardTab.tsx`, `ItemListTab.tsx`.
