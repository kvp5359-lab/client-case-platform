# Документы: сквозная табличная вёрстка строк + доработки источников

**Дата:** 2026-07-09
**Тип:** feat/polish (документы; фронт)
**Статус:** деплой push в main → CI/CD blue/green

---

Продолжение фичи источников Google Drive (см.
[2026-07-09-document-sources-google-drive.md](./2026-07-09-document-sources-google-drive.md)).
Серия UI-правок вёрстки строк документов и файлов источника + функциональные
доработки.

## 1. Сквозная табличная вёрстка строк

Строки документов набора (`DocumentItem`) и файлов источника (`KitSourceFileRow`)
приведены к единой табличной сетке:

- размер и дата — в **отдельных колонках**, прижаты к правому краю (раньше размер
  был внутри строки, даты не было вовсе);
- **выравнивание по вертикали**: все ячейки `align-middle`, meta-спаны через
  `flex` (не `inline-flex` — тот центрировал текст на 1.5px иначе, замерено в
  браузере); имя, размер, дата на одной линии;
- **вертикальные светло-серые разделители** (`border-l gray-100`) перед размером
  и между размером и датой — короче высоты строки (~4px разрывы сверху/снизу);
- **горизонтальные разделители** между строками (`border-b gray-100`);
- размер/дата единого стиля: `12px`, `tabular-nums`, база `gray-400`;
- **умная дата** (`formatSmartDate`: сегодня / вчера / день недели / «30 июн»);
- подсветка размера по порогам шаблона проекта — теперь и у файлов источника.

## 2. Быстрое сжатие PDF из колонки размера

При наведении на размер несжатого **PDF** — иконка сжатия в слоте слева
(запускает то же сжатие, что пункт меню). Во время сжатия — анимация
(бегущий мини-бар) **в колонке размера, вместо числа**. Зелёная иконка «сжат» —
в том же слоте слева. Слот фиксированной ширины, не двигает размер.

## 3. Кнопки и перетаскивание в лотке источника

Файлы источника в лотке набора: кнопки на hover — «Открыть» (в Drive),
«Скрыть» (глаз), «Принять» (в папку). Строки лотка **перетаскиваемые** в любую
папку/слот/на документ набора (тот же MIME-контракт, что у правой панели).
Имена файлов источника — светло-серые (`gray-400`), визуально отделены от
документов.

## 4. Пустые папки — компактно

Вместо большого блока «Нет документов» с воздухом — **лёгкий серый бейдж
«нет документов»** сразу после названия папки. При перетаскивании файла
появляется тонкая пунктирная зона «Перетащите документ сюда». Убраны
разделительные линии над заголовком папки и над блоками «Из источника».

## 5. Стиль бейджа «Статус не выбран»

Белый фон с серой рамкой → просто светло-серый фон (`bg-gray-100`, hover
`gray-200`), без рамки.

## 6. Синхронизация источников

- Пункт **«Обновить файлы из источников»** в верхнем меню «···» — проверяет
  новые/удалённые файлы во **всех** источниках проекта (сводка тостом).
- Кнопка синхронизации ↻ на заголовке каждого источника в правой панели.
- Меню набора «Обновить файлы из источника» — только его источник.

## 7. Фиксы

- **«Без источника» при заполненном `source_id`**: маппинг `fetchSourceDocuments`
  (правая панель) не переносил `sourceId` из БД — исправлено (данные были верны).
- **Пропадал доступный файл источника**: запрос «использованных» файлов не
  отсекал удалённые документы (`is_deleted`) — принятый-затем-удалённый файл
  числился used и исчезал из лотка/панели. Теперь учитываются только живые
  документы (`getSourceDocumentsByProject` + `getSourceDocumentsByKit`).
