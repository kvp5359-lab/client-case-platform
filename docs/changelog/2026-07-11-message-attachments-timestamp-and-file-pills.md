# Отображение вложений в баблах: время оверлеем на вложении + компактные плашки файлов

**Дата:** 2026-07-11
**Тип:** style (фронт, мессенджер)
**Статус:** деплой (push в main → CI/CD blue/green)

---

Серия правок отображения файлов и времени в баблах сообщений — по фидбеку в
процессе. Канальной логики (dispatch/webhook/send/visibility) не касалось,
только визуал ленты.

## Время сообщения — единая логика показа

Раньше время у файловых сообщений жило по-разному (в строке текста, absolute
под баблом, в плашке) и оставляло пустой «подвал». Свели к правилу:

- **Есть картинка** → время оверлеем в правом нижнем углу последней картинки
  (как было).
- **Есть файл(ы), нет картинок** → время оверлеем в правом нижнем углу
  **последнего файла** (новое; раньше — в строке/absolute).
- **Только текст** → время в конце текста.
- **Аудио без текста** → absolute внизу бабла (у плеера углового места нет).

Реализация:
- `MessageBubble` — переменные `hasAudio`/`hasFiles`/`fileOverlayTimestamp`;
  `showTimestamp` для `BubbleTextContent` теперь `!hasImages && !fileOverlayTimestamp`
  (в текст время идёт только без вложений). `fileTimestamp` передаётся обёрнутым
  в ту же полупрозрачную подложку (`timestampPillBg` + `backdrop-blur`), что у
  картинок. absolute-время внизу и резерв `pb-6` оставлены только для аудио.
- `BubbleTextContent` — проп `hasAttachments` заменён на `showTimestamp` (влияет
  и на layout: `!showTimestamp && 'block'`).
- `MessageAttachments` — `fileTimestamp` отдаётся ПОСЛЕДНЕМУ файлу (было — только
  единственному). Проп `flushTop` убирает верхний `mt` у первого блока вложений.
- `FileAttachment` — проп `timestamp` рендерится оверлеем `absolute bottom-1 right-1`
  (контейнер стал `relative`). У файла с бейджем времени меню «…» поднято
  `self-start`, чтобы не пересекалось с временем.

## Плашки файлов и бабл

- Плашка файла — двустрочная (название сверху, размер снизу), иконка `h-5 w-5`,
  меню «…» горизонтальное (`MoreHorizontal`).
- Бабл **только из вложений** (без текста) — маленький равномерный padding
  вокруг (`p-2.5` вместо `px-4 py-2.5`), верхний `mt` первого блока убран
  (`flushTop`), чтобы отступы сверху/снизу/слева/справа были одинаковые.

## Проверки

- tsc 0, eslint 0 по всем изменённым файлам.
- Смок за пользователем после деплоя: один файл / несколько файлов / текст+файл /
  картинка / аудио — время в правильном месте, пустоты под баблом нет, меню и
  время не пересекаются.
