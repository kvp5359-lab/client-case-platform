# Анкеты: ширина «Треть» теперь сохраняется и работает для многострочных полей

**Дата:** 2026-06-10
**Тип:** fix
**Статус:** completed

---

## Проблема

В редакторе шаблона анкеты для поля выбиралась ширина «Треть», но многострочные
поля (`textarea`) всё равно отображались как «половина» — два в ряд с переносом
третьего на новую строку.

**Корень:**
- Ширина «Треть» (`'1/3'`) — это дефолт, и её **намеренно не сохраняли** в
  `options` (оптимизация: «отсутствие = треть»). В данных `width = null`.
- Но у `textarea` пустая ширина включает легаси-поведение: подряд идущие
  многострочные поля авто-группируются в **2 колонки** (половина + половина).
- Итог: выбор «Треть» игнорировался — UI показывал «Треть», рендер делал
  «половину».

## Решение

- [`FormTemplateEditorPage.tsx`](../../src/components/templates/FormTemplateEditorPage/FormTemplateEditorPage.tsx):
  ширина теперь пишется в `options` **всегда**, включая `'1/3'`. Явный `width`
  выводит `textarea` из легаси-авто-группировки, и выбор ширины применяется.
- Данные: трём полям шаблона/анкет («Место работы», «Письма о намерении»,
  «Сайт/соцсети») проставлен `options.width = '1/3'` напрямую в проде, чтобы
  заработало без ручного пересохранения.

**Побочка (ожидаемая):** теперь любое `textarea`, у которого в редакторе открыть
и сохранить ширину, выйдет из авто-парной группировки и встанет по выбранной
ширине. Легаси-группировка остаётся только для полей, где ширину ни разу не
трогали (`width` отсутствует).
