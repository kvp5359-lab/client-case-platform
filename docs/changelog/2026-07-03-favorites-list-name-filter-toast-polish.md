# Избранное (порядок/иконки/префикс), фильтр по названию в списках, полировка тоста

**Дата:** 2026-07-03
**Тип:** feat + fix (UI) + fix (Edge, параллельная сессия)
**Статус:** completed (фронт ждёт деплоя; Edge `telegram-send-message` уже в проде)

---

## 1. Тост нового сообщения — цвет контура и верхняя строка (`MessageToastContent`)

- **Фикс цвета контура (баг).** Карта `ACCENT_BORDER` не была синхронизирована с
  расширенной палитрой (17 акцентов). Для новых цветов (`brown`, `taupe`, `red`,
  `black`, `graphite`, `sky`, `green`) срабатывал фолбэк `border-blue-400` → у
  коричневого треда контур был синим. Добавлены все недостающие цвета; в
  комментарии — пометка держать синхронно с `threadConstants.ts`.
- **Верхняя строка переработана.** Раньше: **Проект** *(Тред)*, отправитель —
  отдельной строкой ниже. Теперь одна строка: **имя отправителя** обычным
  размером, затем в скобках мелким шрифтом проект (**чёрным**) и тред (**в цвет
  акцента треда**, разделитель `·`). У личного диалога без проекта — только тред.
  Отдельная строка с отправителем убрана.
- **Прямой 1:1 диалог — без дубля в скобках.** У прямого чата (WhatsApp / Email /
  TG Business / MTProto) имя треда = имя контакта = отправитель, из-за чего в
  скобках выходил дубль (`София (Sofya) (София (Sofya))`). Теперь для прямого
  чата тред в скобках не показывается: есть проект → только проект `(Клиенты)`;
  нет проекта → скобок нет вовсе. Определение прямого чата (`useNewMessageToast`):
  иконка канала `mail`/`whatsapp` либо совпадение имени треда с отправителем (у
  TG Business/MTProto тред назван по контакту). Групповые/проектные треды с
  осмысленным именем не затронуты.

## 2. Списки: фильтр колонки «Название» по вхождению (`ItemListsPage`)

- Быстрый фильтр заголовков колонок раньше умел только `enum` — чекбоксы
  distinct-значений (для «Статус»/«Проект»/«Исполнители»). Для «Название» такой
  бессмыслен (имена уникальны) → значка фильтра не было.
- Добавлен второй вид быстрого фильтра — **`text`** (поиск по вхождению
  подстроки, регистронезависимо). Колонка «Название» получила текстовый фильтр в
  списках задач/тредов **и** проектов.
- `useQuickFilters` теперь ведёт два состояния (enum-чекбоксы + text-запросы),
  `ColumnFilterMeta` — union `enum | text`, в `TableShell` добавлен
  `ColumnTextFilterPopover` (поле ввода + сброс). Между колонками — AND, поверх
  фильтра самого списка; состояние сессионное.
- **Оговорка:** фильтр клиентский, поверх уже загруженных страниц (список
  подгружается по скроллу) — видит только загруженные строки. Серверный поиск по
  названию — отдельная задача при необходимости.

## 3. Избранное (`SidebarFavoritesButton`)

- **Кнопка «Добавить в избранное» для проекта заработала.** URL проекта — с
  коротким id (`/projects/22`), а код распознавал проект только по полному UUID →
  на странице проекта кнопки «Добавить текущую страницу» не было. Теперь короткий
  id резолвится в UUID по списку проектов (как у досок). `useWorkspaceProjects`
  дополнен полем `short_id`.
- **Порядок групп:** «Треды и задачи» теперь выше «Проектов».
- **Иконки тредов** красятся в акцентный цвет своего треда (загружается
  `accent_color`, класс из `COLOR_TEXT`). Без акцента — серый.
- **Иконки проектов** — как в сайдбаре: `getProjectIcon` по шаблону + цвет
  (fixed/по статусу) через `useProjectIconResolver`; плюс **приглушённый префикс
  имени** (напр. «СР: Мария Орлова»). Реиспользуются те же query-ключи, что и
  сайдбар — без доп. запросов.
- **Новый элемент добавляется в конец группы, а не вверх.** Раньше у всех
  `sort_order = 0`, вторичная сортировка `created_at desc` поднимала новый вверх.
  Теперь при добавлении `sort_order = max(группы) + 1`; оптимистичная строка тоже
  кладётся вниз.

## 4. Отправка вложений в беспроектный групповой чат — 403 (Edge, параллельная сессия)

Вкоммичено заодно, **уже задеплоено** в прод. Разбор — в
[`messenger-ledger.md`](../../.claude/rules/messenger-ledger.md) (запись
2026-07-03).

- **Симптом:** сообщение с файлом в группу уровня воркспейса (`project_id=NULL`)
  зависало в `pending`, в Telegram не приходило (edge `telegram-send-message`
  отдавал 403).
- **Корень:** проверка доступа при вызове с фронта резолвила воркспейс только
  через `project → workspace` и требовала `project_id`; у беспроектного чата его
  нет → 403 до отправки.
- **Фикс:** воркспейс резолвится по проекту **или** из самого сообщения
  (`project_messages.workspace_id` по `message_id`), затем
  `checkWorkspaceMembership`. Зеркало орфан-фикса RLS.

## Грабли

- `ACCENT_BORDER` в тосте — ещё одна точка синхронизации палитры (не входила в
  список «13 точек»); держать вместе с `threadConstants.ts`.
- Быстрый фильтр колонки «Название» — клиентский, по загруженным страницам.
- Иконка/префикс проекта в избранном — те же ключи, что у сайдбара (общий кэш).
- Любой доступ-чек в `*-send`, идущий через `project_id`, ломается на
  workspace-level тредах — резолвить воркспейс из `*.workspace_id`.
