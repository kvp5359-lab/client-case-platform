## №6b — известный латентный баг (НЕ фикшу вслепую)

**Гонка глобального `_botToken` в `telegram-webhook-v2` (§G10).** `sendMessage/editMessage/answerCallback/реакции` в `tg-api.ts` берут токен из мутабельной глобали. При 2+ ботах в одной группе параллельный запрос может перетереть токен между `await`'ами. Скачивание вложений уже исправлено (токен из `ctx.botToken`); отправка — нет.

**Почему не чиню прямо сейчас:** это правка в самой хрупкой зоне, а проверить её можно только живым смоком на группе с двумя активными ботами. Слепая правка карантина без проверки может внести баг хуже исходного (редкого) — это против цели «устойчивость». **Фикс-подход** (когда будет смок-окно): протащить `ctx.botToken` в сигнатуры `tg-api.ts` и всех вызывающих (как сделано для media/upload-slot), убрать `getBotToken()/setBotToken()` глобаль. Затрагивает `tg-api.ts`, `sync.ts`, `callbacks.ts`, `commands.ts`.
